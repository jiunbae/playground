# Destruction Sandbox (부숴볼래?) - 심화 기획문서

---

## 1. 게임 비전

### 한줄 소개 (엘리베이터 피치)
> "물리 엔진 기반 구조물을 체인 리액션으로 파괴하고, 시네마틱 슬로모션으로 감상하고, 숏폼 영상으로 공유하는 파괴 샌드박스 — 부수고, 만들고, 자랑하라."

### 게임의 핵심 감정 (플레이어가 느껴야 할 것)
- **시원함(카타르시스)**: 정교한 구조물이 터치 한 번에 무너지며 스트레스가 날아가는 해방감
- **경탄**: 슬로모션으로 본 파괴의 디테일 — 유리 파편이 햇빛에 반짝이고, 도미노가 연쇄적으로 쓰러지는 "파괴 미학"
- **도전**: "폭탄 1개로 전체 파괴 가능할까?" — 체인 리액션 최적화 퍼즐의 전략적 쾌감
- **자부심**: 내가 만든 시네마틱 파괴 영상이 SNS에서 조회수를 올릴 때의 크리에이터 만족감
- **창작의 기쁨**: 크래프트 모드에서 미니어처를 하나하나 만들어가는 소소한 즐거움

### 디자인 철학 (왜 이 게임이 존재하는가)
Destruction Sandbox는 "파괴"라는 보편적 욕구를 모바일에서 가장 아름답고 정교하게 실현하는 게임이다. 핵심 철학은 두 가지다. 첫째, **파괴를 단순한 난동이 아닌 "예술"로 승격**시킨다 — 시네마틱 카메라, ASMR 사운드, 체인 리액션 퍼즐을 통해. 둘째, **게임 자체가 콘텐츠 생산 도구**다 — 모든 파괴와 건설이 자동으로 숏폼 영상이 되어 바이럴을 만들어낸다. "바이럴 최강, 수익화 전환이 관건"인 하이퍼캐주얼 파워하우스로, 배틀패스와 크래프트 모드를 통해 유입된 대규모 무과금 유저를 수익으로 전환하는 것이 핵심 과제다.

### 타겟 유저 프로필

**1차 타겟 (코어)**
- P1 김하율형 (15~22세 학생): 숏폼 바이럴의 핵심 드라이버. 틱톡/쇼츠에 파괴 영상 업로드, 친구와 듀오 모드. 배틀패스에 소액 과금
- P5 최동현형 (20~30세 경쟁형 남성): 체인 리액션 최적화 + 랭킹 경쟁 + 배틀패스 매 시즌 구매. 소재 연구소 테크트리 장기 몰입

**2차 타겟**
- P3 박준혁형 (25~40세 시스템 분석가): 소재 조합 물리 최적화, 체인 리액션 수학적 분석에 관심. 배틀패스+소재 부스터 과금 가능
- P2 이서연형/P6 한소율형 (20~40세 여성/창작 유저): 크래프트 모드 독립 사용. 파괴 없이 미니어처 건설과 인스타 공유

**비타겟 (의도적 저공략)**
- P4 정미경형 (40대 주부): 파괴 게임 자체에 대한 구조적 거부감. 크래프트 모드로 접근성 개선(+5점)했으나 메인 유저 될 수 없음. 아이와 함께 하는 크래프트 경험만 기대

## 2. 코어 메커니즘 상세

### 핵심 동사 (플레이어가 하는 행동)
1. **부수다** — 터치/도구로 구조물에 힘을 가해 물리 시뮬레이션 기반 파괴를 실행한다
2. **설계하다** — 체인 리액션 퍼즐에서 소재 상호작용을 분석하고 최적 파괴 경로를 계획한다
3. **만들다** — 크래프트 모드에서 블록/소재를 조합하여 미니어처 구조물을 건설한다
4. **감상하다** — 슬로모션 리플레이를 시네마틱 카메라로 감상하고 편집한다
5. **공유하다** — 파괴/건설 영상을 숏폼으로 자동 편집하여 SNS에 공유한다

### 게임플레이 플로우차트

```
[앱 첫 실행]
    │
    ▼
[미니 파괴 — 즉시 체험] ──── 터치→파괴→슬로모션→공유 (15초)
    │
    ├── "더 깊이 파괴하고 싶다면?"
    │         │
    │         ▼
    │   [체인 마스터 캠페인] ──── 60+ 스테이지 퍼즐
    │
    └── "만들기가 더 좋다면?"
              │
              ▼
         [크래프트 모드] ──── 미니어처 건설

[일반 세션 — 3축 구조]
    │
    ├── 축1: [체인 마스터 캠페인]
    │         │
    │         ▼
    │   스테이지 선택 → 소재 구성 분석 → 도구/위치 결정
    │         → 파괴 실행 → 체인 리액션 전개 → 별점 평가
    │         → 시네마틱 리플레이 감상 → 공유 or 재도전
    │
    ├── 축2: [샌드박스 / 크래프트]
    │         │
    │         ├── 자유 파괴: 구조물 선택 → 도구 자유 사용 → 리플레이
    │         └── 크래프트: 건설 키트 선택 → 블록 배치 → 장식
    │              → 갤러리 보존 or "부숴볼래?" 전송
    │
    └── 축3: [위클리 챌린지 / 듀오 대전]
              │
              ├── 위클리 챌린지: 주간 테마 → 도전 → 영상 제출 → 랭킹
              └── 듀오 모드: 코드 매칭 → 실시간 건설 vs 파괴
                   → 자동 녹화 → 분할 화면 리플레이 → 공유
```

### 조작 방식 (터치 제스처별 매핑)

| 제스처 | 파괴 모드 | 크래프트 모드 | 리플레이 모드 |
|--------|-----------|---------------|---------------|
| **탭** | 충격파 발생 (해당 지점) | 블록 선택/배치 | 재생/일시정지 |
| **스와이프** | 절단면 생성 (방향+속도=각도) | 카메라 회전 | 타임라인 스크럽 |
| **롱프레스** | 에너지 축적 → 집중 폭발 | 블록 복제 | 속도 조절 (0.1x~2x) |
| **핀치** | 압축/팽창력 적용 | 블록 크기 조절 | 줌 인/아웃 |
| **두 손가락 회전** | 중력 방향 변경 | 블록 회전 | 카메라 앵글 변경 |
| **세 손가락 탭** | 도구 퀵 전환 | 되돌리기 | 카메라 프리셋 전환 |
| **흔들기 (Shake)** | 지진 효과 발동 | - | 랜덤 앵글 |

### 난이도 곡선 설계

**체인 마스터 캠페인 기준 (60+ 스테이지)**

```
[챕터 1: 나무 입문] (스테이지 1~15)
  │ 소재: 나무만
  │ 학습: 기본 파괴 + 화염 전이 + 도미노 개념
  │ 난이도: ★☆☆ → ★★☆
  │ 세션: 1~2분/스테이지
  │
[챕터 2: 2소재 조합] (스테이지 16~30)
  │ 소재: 나무 + 얼음
  │ 학습: 소재 간 상호작용 (불→얼음=열 충격 파쇄)
  │ 난이도: ★★☆ → ★★★
  │ 세션: 2~3분/스테이지
  │
[챕터 3: 풀체인] (스테이지 31~45)
  │ 소재: 나무 + 얼음 + 금속
  │ 학습: 3소재 풀체인 (불전이→파쇄→도미노)
  │ 난이도: ★★★ → ★★★★
  │ 세션: 3~5분/스테이지
  │
[챕터 4: 마스터] (스테이지 46~60+)
  │ 소재: 전체 + 특수 소재 (고무/수정/자석)
  │ 학습: 복합 퍼즐 + 특수 조건 + 시간 제한
  │ 난이도: ★★★★ → ★★★★★
  │ 세션: 5~10분/스테이지
  │
[보너스: 마스터 챌린지] (무한)
  │ 프로시저럴 생성 스테이지
  │ 매주 시드 기반 갱신
```

### 세션 설계

| 구분 | 시간 | 활동 | 타겟 상황 |
|------|------|------|-----------|
| **최소 세션** | 30초~1분 | 미니 파괴 (터치→파괴→슬로모→공유) | 수업 쉬는 시간, 대기 중 |
| **최적 세션** | 3~5분 | 캠페인 스테이지 1~2개 클리어 + 리플레이 공유 | 출퇴근, 점심시간 |
| **확장 세션** | 10~20분 | 캠페인 연속 진행 + 소재 연구소 + 위클리 챌린지 | 저녁 여가 |
| **크래프트 세션** | 15~30분 | 미니어처 건설 + 촬영 + 공유 | 주말 오후 |
| **소셜 세션** | 5~10분 | 듀오 모드 1~3판 (3분/판) | 친구와 함께 |

## 3. 게임 모드별 상세

### 모드 1: 체인 마스터 캠페인 (메인 모드)
- **목적**: 체인 리액션 퍼즐의 체계적 학습과 진행. 캐주얼에서 미드코어로의 자연 전환 경로
- **규칙**: 각 스테이지에 배치된 소재 구성 분석 → 제한된 도구/횟수로 최대 파괴율 + 체인 달성
- **승리 조건**: 스테이지 클리어 (파괴율 70%+) / 별점 ★(클리어) ★★(체인 3회+) ★★★(최소 입력+풀체인)
- **스코어링**: 파괴율(기본) x 체인 횟수(콤보 배율) x 소재 다양성(조합 배율). "완벽한 체인"(1회 입력 전체 파괴) 시 골드 프레임
- **타겟**: P1 (직관적 퍼즐), P5 (별점 수집 + 최적화), P3 (수학적 분석)

### 모드 2: 샌드박스 (자유 파괴 모드)
- **목적**: 규칙 없는 자유로운 파괴. 스트레스 해소와 실험의 공간
- **규칙**: 없음. 모든 도구/소재 사용 가능, 중력 조절 가능, 시간 무제한
- **승리 조건**: 없음. 파괴 자체가 목적. 리플레이를 통한 감상과 공유가 동기
- **타겟**: P1 (즉시 재미), P5 (실험), P2/P6 (가끔 스트레스 해소)

### 모드 3: 크래프트 모드 (건설 전용 모드)
- **목적**: 파괴와 완전 분리된 건설/꾸미기 경험. 파괴 거부감 유저의 진입점
- **규칙**: 건설 키트(미니어처 하우스/판타지 성/정원/도시) 선택 → 블록/장식 자유 배치
- **승리 조건**: 없음. 갤러리 보존, 사진 촬영(틸트시프트 필터), 공유가 동기
- **선택적 연결**: "이 작품을 부숴볼래?" → 파괴 모드로 전송 가능 (강제 아님)
- **타겟**: P2 (인테리어 감성), P4 (아이와 함께), P6 (디오라마 창작)

### 모드 4: 듀오 모드 (실시간 공방전)
- **목적**: 실시간 건설 vs 파괴 대전으로 소셜 경쟁 + 숏폼 콘텐츠 생성
- **규칙**: 3분 시간 제한. 빌더는 최대한 높이/넓게 쌓기, 파괴자는 최대한 무너뜨리기
- **승리 조건**: 빌더: 최종 잔존 구조물 비율 / 파괴자: 최종 파괴율. 역할 교대 후 합산
- **매칭**: 코드 기반 (서버 불필요, P2P 또는 같은 기기 로컬)
- **타겟**: P1 (쉬는 시간 대전), P5 (경쟁 + 클립 생성)

### 모드 5: 위클리 바이럴 챌린지 (커뮤니티 모드)
- **목적**: 매주 테마 챌린지로 자발적 바이럴 이벤트화
- **규칙**: 주간 테마 ("도미노 최장 기록" / "1초 전멸" / "가장 아름다운 파괴") → 도전 영상 제출
- **승리 조건**: 좋아요/공유 기반 주간 TOP 10 → 다음 주 프리미엄 도구 무료 해금 + 프로필 배지
- **타겟**: P1 (숏폼 콘텐츠 소재), P5 (기록 경쟁)

### 모드 간 관계 (계층 구조)

```
[핵심] 체인 마스터 캠페인 ─── 게임의 메인 루프, 대중적 어필
  │
  ├── [서브] 샌드박스 ─── 자유 파괴, 스트레스 해소
  │
  ├── [서브] 크래프트 모드 ─── 파괴와 분리된 건설 경험 (유저 확장)
  │
  └── [보너스] 위클리 챌린지 / 듀오 대전 ─── 소셜 + 바이럴 엔진
```

### 모드별 타겟 페르소나 매핑

| 모드 | P1 김하율 | P2 이서연 | P3 박준혁 | P4 정미경 | P5 최동현 | P6 한소율 |
|------|:---------:|:---------:|:---------:|:---------:|:---------:|:---------:|
| 체인 마스터 | ★★★ | ★ | ★★★ | - | ★★★★ | ★ |
| 샌드박스 | ★★★ | ★ | ★★ | - | ★★ | ★ |
| 크래프트 | ★ | ★★★ | ★ | ★★ | - | ★★★ |
| 듀오 모드 | ★★★★ | - | ★ | - | ★★★★ | - |
| 위클리 챌린지 | ★★★★ | ★ | ★★ | - | ★★★★ | ★ |

## 4. 콘텐츠 시스템

### 프로시저럴 생성 알고리즘 설계

**구조물 프로시저럴 생성**
- 기본 단위: 빌딩 블록 (벽돌/기둥/보/판) + 소재 속성 (나무/얼음/금속/유리/콘크리트)
- 생성 규칙: 구조 안정성 물리 검증 → 최소 2가지 파괴 경로 보장 → 소재 조합 다양성 확보
- 시드 시스템: 날짜 기반 시드로 동일 구조물 생성 → 위클리 챌린지/친구 대결에 활용
- 난이도 파라미터: 소재 종류 수 (1~6), 구조물 높이 (3~15층), 취약점 수 (1~5), 허용 도구 수 (1~5)

**체인 마스터 스테이지 구성 알고리즘**
```
입력: 챕터 번호 (1~4), 스테이지 번호 (1~15)
출력: 소재 배치 + 사용 가능 도구 + 별점 조건

1. 소재 풀 결정: 챕터 1 = {나무}, 챕터 2 = {나무, 얼음}, ...
2. 구조물 골격 생성: 층 수 = 3 + (스테이지 * 0.5), 폭 = 2 + (스테이지 * 0.3)
3. 소재 배치: 체인 리액션 경로가 최소 1개 존재하도록 소재 인접 배치 보장
4. 도구 할당: 스테이지 전반부 = 3~5개 도구, 후반부 = 1~2개 도구
5. 별점 조건: ★ = 파괴 70%, ★★ = 체인 3회, ★★★ = 최소 입력 + 풀체인
```

**소재 상호작용 그래프**
```
    [나무] ──화염전이──→ [얼음] ──열충격파쇄──→ [유리] ──파편비산──→ [나무/얼음]
      │                    │                                           │
      └──도미노──→ [금속] ──전도──→ [콘크리트] ──충격균열──→ [붕괴]
                     │
                     └──자기가속──→ [자석] ──인력/척력──→ [금속]

    [고무] ──탄성반사──→ [모든 소재] (파편 리바운드)
    [수정] ──공명파쇄──→ [유리/수정] (주파수 기반 연쇄)
    [용암석] ──열방출──→ [나무] (점화), [얼음] (용해)
```

- 결과: 파괴율 비교 → 결과 카드 자동 생성 → 카톡/SNS 공유

**리플레이 코드 교환**
- 캠페인 스테이지 완료 시 "리플레이 코드" 생성 (입력 시퀀스 직렬화)
- 친구가 코드 입력 → 동일 조건 재현 → 결과 비교
- "나보다 잘할 수 있어?" 도발형 공유 유도

### 온보딩/튜토리얼 시퀀스

Destruction Sandbox의 온보딩은 "30초 안에 재미를 느끼게" 설계한다. 텍스트 설명 제로, 즉시 체험형.

```
Step 1: [미니 파괴 — 즉시 체험] (15초)
  화면에 유리 구조물 1개 등장
  → 화면 아무 곳이나 탭 유도 (손가락 애니메이션)
  → 탭: 구조물 산산조각 + 슬로모션 자동 재생
  → ASMR 유리 깨지는 소리 + 햅틱
  → "시원하죠?" 작은 텍스트

Step 2: [두 번째 파괴 — 도구 체험] (15초)
  나무 구조물 등장 + 화염 도구 하이라이트
  → 화염 도구 드래그하여 나무에 적용
  → 불이 번져 전체 소실 + 슬로모션
  → "불이 번지네요!" → 체인 리액션 개념 자연 학습

Step 3: [세 번째 파괴 — 체인 리액션] (20초)
  나무+얼음 복합 구조물 + 화염 도구
  → 화염 → 나무 소실 → 열기가 얼음에 전달 → 얼음 파쇄
  → "체인 리액션!" 텍스트 + 스코어 표시
  → 시네마틱 리플레이 자동 재생

Step 4: [공유 유도] (5초)
  리플레이 하단에 공유 버튼 하이라이트
  → "이 순간을 공유해 보세요" (스킵 가능)

Step 5: [분기 선택] (5초)
  "더 깊이 파괴하고 싶다면 → 체인 마스터"
  "만들기가 더 좋다면 → 크래프트 모드"
  → 선택에 따라 해당 모드로 이동
  → 튜토리얼 종료 (총 60초 이내)
```

  - 콘크리트: 균열 확산 + 붕괴 (저주파 럼블)
- 슬로모션 연동: 속도 감소 시 사운드도 동기 감속 (저음화 + 디테일 강화)

**햅틱 피드백**

- 슬로모션: 부드러운 저진동 (파편 낙하 동기)

**BGM**
- 캠페인: 일렉트로닉 + 오케스트라 하이브리드 (긴장감 + 웅장함)

### 애니메이션 원칙
- **물리 퍼스트**: 모든 파괴 애니메이션은 사전 제작 아닌 실시간 물리 시뮬레이션. 매번 다른 결과
- **슬로모션 강조**: 파괴 클라이맥스 순간 자동 0.3x 감속 + 카메라 자동 줌인
- **체인 리액션 비주얼**: 체인 전파 경로를 에너지 라인(네온 글로우)으로 시각화
- **UI 트랜지션**: 스냅 애니메이션 (빠르고 경쾌한 200ms) — 힐링 게임과 대조적으로 에너지 느낌
- **파괴 파티클**: GPU 파티클 시스템으로 대량 파편 (최대 5,000개) 처리. 파편이 컬러풀하게 빛남
- **크래프트 모드**: 블록 배치 시 "톡" 소리 + 부드러운 스냅 인 애니메이션 (레고 조립 느낌)

### 엔진/프레임워크 추천

| 영역 | 기술 | 선정 근거 |
|------|------|-----------|
| 게임 엔진 | **Unity 2022 LTS + URP** | 모바일 물리 최적화 우수, VFX Graph 파티클, 크로스플랫폼 |

| 파괴 시스템 | **커스텀 Voronoi Fracture + 사전 연산** | 소재별 프랙처 패턴 사전 계산 → 런타임에 인스턴싱 |
| 파티클 | **VFX Graph (GPU 파티클)** | 대량 파편(5,000+) GPU 처리로 CPU 부하 최소화 |

### 핵심 기술 과제

1. **모바일 물리 시뮬레이션 성능**: 수백 개 리지드바디의 실시간 물리 연산이 30fps 유지 필요. DOTS Physics의 ECS 기반 병렬 처리 + LOD(원거리 파편 단순화) + 파편 수 동적 제한(기기 성능 감지 → 최대 파편 수 자동 조절)
2. **Voronoi Fracture 사전 연산**: 런타임 프랙처는 프레임 드롭 유발. 모든 소재 블록의 파쇄 패턴을 빌드 타임에 사전 연산하여 메시 캐시로 저장. 런타임에는 인스턴싱만 수행
3. **리플레이 시스템의 결정적 재현**: 물리 시뮬레이션은 부동소수점 오차로 비결정적일 수 있음. 모든 리지드바디의 Transform+Velocity를 매 프레임 기록(프레임당 약 50바이트/오브젝트)하여 정확한 재현 보장
4. **듀오 모드 P2P 동기화**: WebRTC 기반이나 물리 시뮬레이션의 동기화 필요. 빌더 디바이스가 호스트 → 물리 상태를 스트리밍 → 파괴자 디바이스는 렌더링만 (권위적 서버 모델의 P2P 변형)
5. **AI 자동 편집 알고리즘**: "최고 장면" 자동 감지를 위해 프레임별 "임팩트 스코어" 계산 (파편 수 변화량 + 속도 최대값 + 체인 발생 여부). 상위 3개 하이라이트 구간을 자동 컷 편집

### 오프라인/서버리스 아키텍처

```
[디바이스 로컬]
    │
    ├── 물리 엔진 ──── Unity DOTS Physics (실시간 시뮬레이션)
    │
    ├── 프로시저럴 엔진 ── 시드 기반 구조물 생성 (날짜 → 시드 → 구조물)
    │
    ├── 리플레이 시스템 ── 물리 상태 기록 → 재생 → 시네마틱 편집
    │
    ├── AI 편집 엔진 ── 프레임별 임팩트 스코어 → 하이라이트 감지 → 자동 컷
    │
    ├── 배틀패스 엔진 ── 시즌 시작일 기반 로컬 계산 + 미션 추적
    │
    ├── 소재 연구소 ── 연구 포인트 로컬 관리 + 해금 상태 저장
    │
    ├── 공유 시스템 ── NatCorder MP4 생성 → OS Share Sheet
    │
    ├── 듀오 모드 ── WebRTC P2P (코드 기반 매칭, STUN/TURN 서버만 필요)
    │     └── 로컬 멀티플레이: 같은 기기 화면 분할 (서버 완전 불필요)
    │
    └── 클라우드 백업 ── iCloud / Google Drive (선택적)
```

- 서버 비용: 듀오 모드 원격 매칭용 STUN/TURN 서버만 (월 약 $10~$30, 최소 비용)
