# Draw Alive (그려서 살려내기) - 심화 기획문서

---

## 1. 게임 비전

### 한줄 소개 (엘리베이터 피치)
> "네 낙서가 살아 움직인다 — 그려서 만들고, 굴리고, 날리고, 공유하라."

### 게임의 핵심 감정
- **경이(Wonder)**: 내가 대충 그린 그림이 살아 움직이는 순간의 놀라움
- **창조의 기쁨(Creative Joy)**: 나만의 방식으로 문제를 해결했다는 성취감
- **공유의 쾌감(Sharing Delight)**: 내 엉뚱한 결과물을 보여주고 싶은 충동

### 디자인 철학
Draw Alive는 "형태가 기능을 결정한다"는 하나의 원칙 위에 서 있다. 플레이어가 그린 모든 선은 물리적 의미를 갖고, 그 결과는 예측 불가능하면서도 논리적이다. 그림 실력이 아닌 물리적 사고가 보상받는 게임이며, 어떤 수준의 낙서든 유효한 게임 오브젝트가 된다. 이 게임이 존재하는 이유는 "창작의 민주화" — 누구나 30초 만에 창작자가 되어, 자신만의 결과물을 세상에 보여줄 수 있는 가장 낮은 진입장벽의 창작 게임을 만드는 것이다.

### 타겟 유저 프로필

| 구분 | 페르소나 | 핵심 동기 |
|------|----------|-----------|
| **1차 타겟** | P6 한소율 (34세, 일러스트레이터) | 창작 도구 + 커뮤니티 + 예술적 표현 |
| **1차 타겟** | P1 김하율 (18세, 고등학생) | 숏폼 콘텐츠 생산 + 또래 공유 + 즉각적 재미 |
| **1차 타겟** | P4 정미경 (43세, 주부) | 가족 함께하기 + 교육적 가치 + 힐링 |
| **2차 타겟** | P2 이서연 (26세, 직장인) | 출퇴근 자투리 시간 + 감성 콘텐츠 소비 + SNS 공유 |
| **2차 타겟** | P3 박준혁 (32세, 개발자) | 물리 속성 최적화 + 효율 리더보드 + 전략 깊이 |
| **비타겟** | P5 최동현 (22세, 대학생) | 실시간 PvP 경쟁 중심 — 서브 게임으로는 가능하나 메인은 불가 |

## 2. 코어 메커니즘 상세

### 핵심 동사 (플레이어가 하는 행동)
1. **그린다 (Draw)** — 터치로 자유롭게 형태를 그린다
2. **살린다 (Animate)** — 그린 그림이 물리 오브젝트로 변환되어 움직인다
3. **풀어낸다 (Solve)** — 물리 법칙을 활용해 퍼즐 목표를 달성한다
4. **수집한다 (Collect)** — 발견한 물리 레시피와 창작물을 도감에 모은다
5. **공유한다 (Share)** — 결과물을 GIF/카드로 만들어 SNS에 보여준다

### 게임플레이 플로우차트

```
[앱 실행]
    │
    ▼
[첫 실행?] ─── Yes ──→ [온보딩: 빈 캔버스]
    │                        │
    No                       ▼
    │                   [자유 드로잉 (5~10초)]
    ▼                        │
[메인 메뉴]                  ▼
    │                   [자동 물리 시뮬레이션 → "마법의 순간!"]
    ├─→ [챌린지 모드]        │
    │       │                ▼
    │       ▼           [코어 루프 진입]
    │   [스테이지 선택]
    │       │
    │       ▼
    │   [목표 확인] ←──────────────────┐
    │       │                          │
    │       ▼                          │
    │   [드로잉 페이즈]                │
    │       │                          │
    │       ▼                          │
    │   [시뮬레이션 실행]              │
    │       │                          │
    │       ├── 성공 ──→ [결과 화면]   │
    │       │               │          │
    │       │               ├─→ [GIF 공유]
    │       │               ├─→ [도감 저장]
    │       │               └─→ [다음 스테이지]
    │       │                          │
    │       └── 실패 ──→ [리트라이] ───┘
    │
    ├─→ [샌드박스 모드] ──→ [자유 창작 + 실험]
    │
    ├─→ [커뮤니티 허브] ──→ [갤러리 / 대회 / 팔로우]
    │
    ├─→ [2인 대결] ──→ [로컬 분할 / 온라인 비동기]
    │
    └─→ [키즈 모드] ──→ [연령별 난이도 / 물리 실험]
```

### 조작 방식 (터치 제스처별 매핑)

| 제스처 | 기본 모드 | 한 손 모드 (도형 팔레트) |
|--------|-----------|------------------------|
| **한 손가락 드래그** | 자유 드로잉 (선 그리기) | 선택된 도형 배치 위치/크기 조절 |
| **탭** | 잉크 한 방울 점 찍기 | 도형 팔레트에서 도형 선택 |
| **두 손가락 핀치** | 캔버스 줌 인/아웃 | 배치된 도형 크기 조절 |
| **두 손가락 회전** | 캔버스 회전 | 배치된 도형 회전 |
| **길게 누르기** | 지우개 모드 활성화 | 도형 삭제 |
| **두 번 탭** | 실행 취소 (Undo) | 실행 취소 |
| **세 손가락 탭** | 시뮬레이션 실행 | 시뮬레이션 실행 |
| **좌측 가장자리 스와이프** | 색상 팔레트 열기 | 자유 드로잉 ↔ 팔레트 모드 전환 |

### 난이도 곡선 설계

```
난이도
  │
  │                                    ╱ 물리 마스터 챌린지
  │                                  ╱   (10단계 누적 제약)
  │                              ╱──╱
  │                          ╱──╱
  │                  ╱── ╱──╱  복합 목표 + 물리 조합
  │              ╱──╱
  │          ╱──╱  색상→속성, 잉크 제한
  │      ╱──╱
  │  ╱──╱  자유 드로잉 + 단순 목표
  │╱ 온보딩 (마법의 순간)
  └──────────────────────────────────── 스테이지
  1   5   10      25      50     100+

  [적응형 난이도 조절]
  - 연속 3회 실패 → 잉크 +20% 보너스 + 힌트 제공
  - 연속 3회 3스타 → 보너스 제약 조건 제안 (선택)
```

### 세션 설계

| 세션 유형 | 시간 | 상황 | 활동 |
|-----------|------|------|------|
| **최소 세션** | 1~2분 | 신호 대기, 잠깐의 틈 | 챌린지 1스테이지 or 샌드박스 낙서 |
| **최적 세션** | 5~10분 | 출퇴근, 쉬는 시간 | 챌린지 3~5스테이지 + 결과 공유 |
| **몰입 세션** | 20~30분 | 여유 시간 | 물리 마스터 챌린지 + 커뮤니티 + 대결 |
| **가족 세션** | 15~20분 | 가족 시간 | 키즈 모드 + 함께하기 + 물리 실험 |

## 3. 게임 모드별 상세

### 3-1. 챌린지 모드 (핵심 모드)

- **목적**: 물리 퍼즐을 드로잉으로 풀어내는 코어 경험 제공
- **규칙**: 잉크량 제한 내에서 오브젝트를 그려 스테이지 목표 달성
- **승리 조건**: 주어진 목표 달성 (별 획득, 경로 연결, 오브젝트 이동 등)
- **평가**: 잉크 사용량, 클리어 시간, 보너스 조건 기반 3스타 평가
- **타겟 페르소나**: P1 김하율, P2 이서연 (접근성), P3 박준혁 (깊이)

### 3-2. 샌드박스 모드 (핵심 모드)

- **목적**: 제한 없는 자유 창작과 물리 실험
- **규칙**: 잉크량 무제한, 목표 없음, 다중 오브젝트 동시 생성
- **승리 조건**: 없음 (자유 놀이)
- **주요 활동**: Rube Goldberg Machine 창작, 물리 레시피 발견, 아트워크 제작
- **타겟 페르소나**: P6 한소율 (창작), P4 정미경 (아이와 놀이)

### 3-3. 2인 드로잉 대결 모드 (확장 모드)

- **목적**: 경쟁을 통한 리플레이성과 소셜 동기 확보
- **규칙**: 동일 스테이지 동시 풀이, 속도 40% + 완성도 60% 가중치
- **승리 조건**: 종합 점수 우위
- **방식**: 로컬(화면 분할/턴제) 또는 온라인(비동기 실시간)
- **타겟 페르소나**: P5 최동현 (경쟁), P1 김하율 (친구 대결)

### 3-4. 주간 창작 대회 (확장 모드)

- **목적**: 커뮤니티 참여와 바이럴 콘텐츠 생성
- **규칙**: 주간 테마에 맞춘 클리어 데이터 제출, 커뮤니티 투표
- **승리 조건**: 투표 수 상위 TOP 10 선정
- **보상**: 한정 펜/브러시, 명예의 전당 등재
- **타겟 페르소나**: P6 한소율, P1 김하율

### 3-5. 가족/키즈 모드 (보조 모드)

- **목적**: 가족 시장 진입 + 교육적 가치 제공
- **규칙**: 연령별 난이도(6-8세/9-12세/13세+), 물리 실험 스테이지
- **승리 조건**: 스테이지 클리어 + "물리 발견 카드" 수집
- **안전**: 온라인 기능/채팅 비활성화, 광고 자동 숨김
- **타겟 페르소나**: P4 정미경 + 자녀

### 3-6. 물리 마스터 챌린지 (확장 모드)

- **목적**: 하드코어 유저의 장기 목표 + 전략 깊이 제공
- **규칙**: 동일 스테이지에 10단계 누적 제약 (잉크 감소, 시간 제한, 금지 색상 등)
- **승리 조건**: 제약 조건 하에서 스테이지 클리어
- **보상**: 단계별 전용 보상 + 커뮤니티 배지
- **타겟 페르소나**: P3 박준혁

### 모드 간 계층 관계

```
[코어 계층]
  ├── 챌린지 모드 (주요 진행축)
  └── 샌드박스 모드 (자유 창작축)

[확장 계층] — 코어를 충분히 경험한 후 자연스럽게 노출
  ├── 2인 대결 모드
  ├── 주간 창작 대회 / 커뮤니티 허브
  └── 물리 마스터 챌린지

[보조 계층] — 특정 타겟을 위한 독립 진입점
  └── 가족/키즈 모드
```

## 4. 콘텐츠 시스템

### 프로시저럴 생성 알고리즘 설계

**스테이지 생성 파이프라인**:
```
1. 시드 결정 (스테이지 번호 또는 날짜 해시)
   │
2. 지형 템플릿 선택 (5종 기본 레이아웃 중 시드 기반 선택)
   │
3. 장애물 배치 (파라미터: 밀도, 높이, 간격 — 시드 기반 랜덤)
   │
4. 목표 배치 (별/타겟/도착점 — 지형 대비 최소 거리 보장)
   │
5. 잉크량 산정 (최적 경로 대비 1.5~2.0배)
   │
6. AI 검증 시뮬레이션 (3종 전략으로 클리어 가능성 테스트)
   │
7. 합격 → 유저 제공 / 불합격 → 시드+1로 재생성
```

**난이도 스케일링 공식**:
- 장애물 수 = base(3) + floor(스테이지 / 8)
- 잉크량 배수 = max(1.2, 2.0 - 스테이지 * 0.008)
- 복합 목표 수 = min(3, 1 + floor(스테이지 / 20))

### 콘텐츠 수량 계획

| 시점 | 핸드크래프트 스테이지 | 프로시저럴 | 물리 레시피 | 펜/브러시 | 물리 실험 |
|------|---------------------|-----------|------------|----------|----------|
| **출시 (MVP)** | 50 | 무한 | 30 | 10 (기본 5 + 해금 5) | 10 |
| **3개월** | 80 | 무한 | 60 | 25 | 20 |
| **6개월** | 120 | 무한 | 80 | 40 | 30 |
| **1년** | 150 | 무한 | 100+ | 60 | 40 |

### 시즌/이벤트 설계 (서버 없이 가능한 방식)

**시즌 구조 (3개월 주기)**:
- 시즌 테마: 봄/여름/가을/겨울 자연 테마
- 시즌별 한정 펜/브러시 5종, 배경 테마 2종
- 시즌 패스를 통한 보상 트랙 (무료 10단계 + 유료 20단계)

**주간 이벤트 (클라이언트 날짜 시드 기반)**:
- 월요일: 주간 테마 챌린지 시작 (시드 = 주차 번호)
- 수요일: 중간 집계 공개 (로컬 시뮬레이션)
- 일요일: 주간 결과 + TOP 10 선정

**일일 이벤트**:
- 일일 챌린지 스테이지 (DATE 시드 기반 자동 생성)
- 일일 물리 실험 (키즈 모드 연계)

## 5. 진행 시스템 & 메타 게임

### 코어 루프 (1세션 단위) 상세 플로우

```
[세션 시작]
    │
    ▼
[스테이지 진입] → 잉크량/목표 확인 (5초)
    │
    ▼
[드로잉 페이즈] → 터치로 오브젝트 그리기 (30초~2분)
    │
    ▼
[시뮬레이션 실행] → 물리 반응 관찰 (10~30초) ← "마법의 순간"
    │
    ├── 성공 ──→ [스타 평가 + 잉크 효율 표시]
    │                │
    │                ├─→ [물리 레시피 발견?] → 도감 등록 팝업
    │                ├─→ [GIF 자동 생성] → 공유 프롬프트
    │                ├─→ [경험치 + 해금 진행]
    │                └─→ [다음 스테이지 자동 전환]
    │
    └── 실패 ──→ [리트라이 선택]
                    ├─→ 즉시 재도전
                    ├─→ 광고 시청 → 잉크 추가
                    └─→ 힌트 보기 (연속 실패 시)
```

### 메타 루프 (장기 진행) 마일스톤 설계

| 마일스톤 | 조건 | 해금 보상 |
|----------|------|----------|
| 첫 번째 마법 | 온보딩 완료 | 자유 펜 3색 해금 |
| 초보 화가 | 10스테이지 클리어 | 간편 도형 팔레트 해금 |
| 퍼즐 마스터 | 25스테이지 클리어 | 샌드박스 모드 확장 |
| 색의 과학자 | 물리 레시피 10개 발견 | 레시피 도감 UI 해금 |
| 도전자 | 50스테이지 클리어 | 2인 대결 모드 해금 |
| 크리에이터 | 작품 10개 갤러리 게시 | 크리에이터 프로필 활성화 |

### 사운드 디자인 방향
- **BGM**: 경쾌한 어쿠스틱 기타 + 목관 악기 루프 (드로잉 중), 긴장감 있는 퍼커션 추가 (시뮬레이션 중)
- **SFX - 드로잉**: 부드러운 크레용 긁는 소리, 색상 변경 시 톤 변화
- **SFX - 시뮬레이션**: 탄성 바운스 "보잉", 충돌 "쿵", 구르기 "드르르", 파괴 "딱깍"
- **SFX - 성공**: 동요풍 짧은 팡파레 (맑은 종소리 + 작은 박수)
- **SFX - 실패**: 코믹한 "삐~" + 부드러운 무너짐 소리 (좌절감 최소화)
- **시뮬레이션 사운드**: 물리 반응에 동기화된 실시간 사운드 (무게감/속도에 따라 pitch 변화)

### 애니메이션 원칙
- **드로잉 → 오브젝트 변환**: 0.3초간 그림 외곽선이 빛나며 "살아나는" 트랜지션
- **물리 시뮬레이션**: 물리 엔진 기반 실시간 (squash & stretch 원칙 적용)
- **UI 전환**: 종이가 넘어가는 듯한 페이지 플립 효과
- **성공 연출**: 별이 튀어나오는 파티클 + 잉크가 퍼지는 수채화 효과
- **캐릭터(오브젝트) 눈깜빡임**: 생성된 오브젝트에 간단한 눈 2개 자동 추가 → 깜빡임 애니메이션으로 "살아있는 느낌" 부여

## 10. 기술 요구사항

### 엔진/프레임워크 추천

| 선택지 | 추천도 | 이유 |
|--------|--------|------|
| **Unity 2D (추천)** | ★★★★★ | Box2D 내장, 크로스플랫폼, 풍부한 2D 에셋/플러그인 생태계 |
| Godot 4 | ★★★★☆ | 경량, 오픈소스, 2D 물리 우수하나 모바일 최적화/커뮤니티 에셋이 Unity 대비 부족 |
| Cocos2d-x | ★★★☆☆ | 경량이나 물리 엔진 커스텀 부담, 한국 커뮤니티 제한적 |

**최종 추천: Unity 2022 LTS + Box2D (Unity Physics 2D)**

### 핵심 기술 과제

| 과제 | 난이도 | 해결 방향 |
|------|--------|----------|
| 자유 드로잉 → 물리 콜라이더 변환 | 높음 | 터치 포인트 → 베지어 곡선 → 폴리곤 메시 → PolygonCollider2D 자동 생성. 폴리곤 수 자동 간소화(Douglas-Peucker 알고리즘) |
| 물리 시뮬레이션 성능 | 중~높음 | 동시 오브젝트 수 제한(최대 20), LOD 시스템, 복잡한 폴리곤 자동 단순화 |
| 물리 엔진 결정론적 재현 | 높음 | Unity Physics 2D의 고정 타임스텝(Fixed Timestep) 사용, 부동소수점 연산 순서 고정, 리플레이 검증 테스트 |
| 자동 GIF 생성 | 중 | RenderTexture 프레임 캡처 → 커스텀 GIF 인코더 (UniGif 활용) |
| 프로시저럴 스테이지 밸런스 | 중 | AI 사전 검증 (3종 휴리스틱 전략으로 시뮬레이션) + 생성 실패 시 재생성 |
| 한 손 모드 도형 팔레트 UX | 중 | 화면 좌측 슬라이드 패널, 도형 프리뷰 + 드래그 배치, 원핸드 리치존 분석 기반 배치 |

### 오프라인/서버리스 아키텍처

```
[클라이언트 (완전 로컬)]
    │
    ├── 게임 로직 (물리 엔진, 스테이지 생성, 점수 계산)
    ├── 데이터 저장 (SQLite + PlayerPrefs)
    ├── GIF/이미지 생성 (RenderTexture 기반)
    ├── 공유 (OS 네이티브 공유 시트)
    └── 딥링크 (URL Scheme — 서버 불필요)

[선택적 네트워크 (기능 확장용)]
    ├── 광고 SDK (AdMob — 오프라인 시 큐잉)
    ├── IAP (스토어 네이티브 — 구매 시만 온라인)
    ├── Analytics (Firebase — 오프라인 이벤트 캐싱 후 전송)
    └── 매칭 서버 (온라인 대결용 — 경량 WebSocket, 선택적)

[커뮤니티 허브 (Tier 3 단계에서 추가)]
    └── 최소 서버 (작품 메타데이터, 투표 수, 팔로우 — Firebase Realtime DB 활용)
```
