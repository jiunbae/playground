# One Hand Fortress (한 손의 요새) - 심화 기획문서

---

## 1. 게임 비전

### 한줄 소개 (엘리베이터 피치)
> "잠든 마을을 깨우는 수호자 — 엄지 하나로, 따뜻한 전략을 펼쳐라."

### 게임의 핵심 감정
- **수호의 따뜻함(Warmth of Protection)**: 작은 마을을 지키고 깨우는 보람과 애정
- **전략적 만족(Strategic Satisfaction)**: 제한된 수호탑 8종으로 최적 배치를 찾아낸 성취감
- **치유(Healing)**: 수채화 세계 속에서 잠든 자연이 깨어나는 아름다운 변화를 지켜보는 평온

### 디자인 철학
One Hand Fortress는 "전쟁"이 아닌 "수호"의 게임이다. 적을 물리치는 것이 아니라 잠든 자연을 깨우는 것이며, 파괴가 아닌 회복을 목표로 한다. 이 게임이 존재하는 이유는 "전략 게임의 감성적 재정의" — 타워디펜스의 전략적 깊이를 유지하면서, 수채화/동화풍 비주얼과 따뜻한 서사로 전혀 다른 감성의 경험을 만드는 것이다. 엄지 하나만으로 조작 가능한 미니멀 디자인은 이 철학의 물리적 구현이다.

### 타겟 유저 프로필

| 구분 | 페르소나 | 핵심 동기 |
|------|----------|-----------|
| **1차 타겟** | P3 박준혁 (32세, 개발자) | 미니멀 전략 깊이 + 효율 최적화 + 비동기 경쟁 |
| **1차 타겟** | P2 이서연 (26세, 직장인) | 수채화 감성 + 출퇴근 한 손 플레이 + 마을 꾸미기 |
| **2차 타겟** | P4 정미경 (43세, 주부) | 마을 가꾸기 힐링 + 탈전쟁 따뜻한 서사 + 가족 협동 |
| **2차 타겟** | P6 한소율 (34세, 일러스트레이터) | 아트 게임 정체성 + 수채화 컬렉션 + 인터랙티브 스토리 |
| **2차 타겟** | P5 최동현 (22세, 대학생) | 비동기 PvP + 주간 토너먼트 + 시즌 랭킹 |
| **비타겟** | P1 김하율 (18세, 고등학생) | 또래 유행 가능성 낮음 — 숏폼 챌린지로 최소 접점 유지 |

## 2. 코어 메커니즘 상세

### 핵심 동사 (플레이어가 하는 행동)
1. **배치한다 (Place)** — 수호탑을 전략적 위치에 탭으로 배치한다
2. **지킨다 (Protect)** — 잠의 파편이 마을에 도달하지 않도록 수호한다
3. **깨운다 (Awaken)** — 모든 잠의 물결을 물리쳐 마을을 잠에서 깨운다
4. **가꾼다 (Nurture)** — 보상으로 얻은 포인트로 마을을 데코레이션한다
5. **나눈다 (Share)** — 배치도 아트워크와 마을을 코드로 공유한다

### 게임플레이 플로우차트

```
[앱 실행]
    │
    ▼
[첫 실행?] ─── Yes ──→ [인터랙티브 튜토리얼 (5스테이지)]
    │                        │
    No                       ▼
    │                   [코어 루프 진입]
    ▼
[메인 메뉴 — "나의 마을" 중심]
    │
    ├─→ [퍼즐 (스테이지 선택)]
    │       │
    │       ▼
    │   [난이도 선택: 이지/노멀/하드]
    │       │
    │       ▼

    │       │ 초기 골드로 수호탑 배치          │
    │       ▼                                 │
    │   [잠의 물결 시작]                      │
    │       │ 잠의 파편 등장                   │
    │       ▼                                 │

    │       │                      ├─→ [깨우기 포인트 획득]
    │       │                      ├─→ [배치도 아트워크 저장]
    │       │                      ├─→ [리플레이 코드 생성]
    │       │                      └─→ [다음 스테이지]
    │       │                                 │
    │       └── 마을 잠듦 ──→ [리트라이] ─────┘
    │
    ├─→ [나의 마을 (데코레이션)]
    │       ├─→ 건물 배치 / 정원 꾸미기

    │       ├─→ 마을 앨범 (성장 기록)
    │       └─→ 마을 코드 공유
    │
    ├─→ [대결/토너먼트]
    │       ├─→ 도전장 보내기/받기

    │       └─→ 시즌 랭킹
    │
    ├─→ [협동 모드]
    │       ├─→ 로컬 (화면 분할)
    │       └─→ 온라인 (비동기 리플레이)
    │
    └─→ [스토리/컬렉션]
            ├─→ 인터랙티브 스토리 비네트
            └─→ 계절 일러스트 컬렉션
```

### 조작 방식 (터치 제스처별 매핑)

| 제스처 | 기능 | 비고 |
|--------|------|------|
| **탭** | 현재 선택된 수호탑을 해당 위치에 배치 | 핵심 조작 — 엄지 하나로 가능 |
| **좌우 스와이프** | 하단 수호탑 슬롯 전환 | 기본 4종 + 특수 4종 |
| **길게 누르기** | 배치된 수호탑 업그레이드 (레벨 1→2→3) 또는 판매 | 팝업 메뉴: 업그레이드/판매 선택 |
| **더블 탭** | 액티브 스킬 발동 (화면 전체 효과) | 쿨타임 있음 |
| **두 손가락 핀치** | 줌 인/아웃 (맵 전체 조망 ↔ 세부 확인) | 선택적 — 한 손으로 불필요 시 자동 줌 |
| **위로 스와이프** | 잠의 물결 배속 (x2) | 여유 시 시간 단축 |
| **아래로 스와이프** | 일시정지 + 전체 맵 조망 | 전략 재검토 시간 |

### 난이도 곡선 설계

```
난이도
  │
  │                                      ╱ 무한 스테이지
  │                                    ╱   (프로시저럴)
  │                              ╱──  ╱
  │                          ╱──╱──╱
  │                      ╱──╱  특수 수호탑 + 보스 패턴
  │                  ╱──╱
  │              ╱──╱  기본 4종 + 다중 경로
  │          ╱──╱
  │      ╱──╱  기본 2종 + 단일 경로
  │  ╱──╱
  │╱ 튜토리얼 (탭 배치만)
  └──────────────────────────────────────── 스테이지

  [이지모드 오버레이]
  - 적 체력/속도 30% 감소
  - 시작 골드 50% 증가

  - 오토배틀 선택 가능
```

### 세션 설계

| 세션 유형 | 시간 | 상황 | 활동 |
|-----------|------|------|------|
| **최소 세션** | 3분 | 버스/지하철 한두 정거장 | 스테이지 1개 클리어 |
| **최적 세션** | 8~12분 | 출퇴근 편도 | 스테이지 2~3개 + 마을 꾸미기 |
| **몰입 세션** | 20~30분 | 여유 시간 | 토너먼트 도전 + 스토리 감상 + 마을 확장 |
| **협동 세션** | 10~15분 | 친구/가족과 함께 | 로컬 협동 2~3스테이지 |

## 3. 게임 모드별 상세

### 3-1. 스토리 퍼즐 모드 (핵심 모드)

- **목적**: 잠든 마을을 월드별로 깨우는 메인 진행축
- **구조**: 5개 월드 x 10스테이지 = 50 핸드크래프트 스테이지 + 프로시저럴 무한
- **규칙**: 골드 관리 → 수호탑 배치 → 잠의 파편 수호 → 마을 깨우기
- **승리 조건**: 모든 잠의 물결을 마을 도달 전에 깨움 (체력 잔여량 기반 스타 평가)
- **월드 서사**:
  - 월드 1: "잠든 숲의 정원" — 겨울잠에 빠진 작은 정원에 봄을 불러오기
  - 월드 2: "호수 마을의 새벽" — 안개에 잠긴 호숫가 마을에 아침 빛을 전하기
  - 월드 3: "달빛 언덕의 약속" — 어둠에 덮인 언덕 위에 별빛을 되돌려주기
  - 월드 4: "눈 내리는 항구" — 얼어붙은 항구에 따뜻한 바람을 보내기
  - 월드 5: "구름 위의 마을" — 구름에 가려진 하늘 마을에 햇살을 비추기
- **타겟 페르소나**: P3 박준혁 (전략), P2 이서연 (감성), P4 정미경 (스토리)

### 3-2. 마을 가꾸기 모드 (핵심 모드)

- **목적**: 퍼즐 외 장기 진행 동기 + 캐주얼 유저 잔류
- **규칙**: 퍼즐 보상 "깨우기 포인트"로 건물/정원/주민을 배치
- **승리 조건**: 없음 (자유 데코레이션, 마을 레벨 마일스톤)
- **콘텐츠**:
  - 건물 20종 (빵집, 꽃집, 등대, 다리, 풍차 등)
  - 자연 요소 15종 (꽃밭, 나무, 연못, 벤치, 분수대 등)
  - NPC 주민 12명 (각각 고유 활동 + 미니 에피소드)
  - 계절 데코 세트 4종 (봄/여름/가을/겨울)
- **타겟 페르소나**: P4 정미경 (마을 가꾸기), P2 이서연 (꾸미기 + 공유)

### 3-3. 비동기 PvP / 토너먼트 모드 (확장 모드)

- **목적**: 경쟁형 유저의 장기 동기 확보
- **도전장 시스템**:
  - 같은 시드 스테이지에서 각자 플레이 → 리플레이 코드 교환 → 결과 비교
  - 비교 지표: 클리어 시간, 잔여 체력, 골드 효율
  - 딥링크로 "이 스테이지 나보다 잘 할 수 있어?" 전송
- **주간 전략 토너먼트**:
  - 매주 지정 시드 5스테이지 → 종합 점수 랭킹
  - 부문별 랭킹: 최소 타워 / 최단 시간 / 최소 자원 / 최고 잔여 체력 / 종합
  - TOP 100 보상: 수채화 타워 스킨, 1위 배치도 "이번 주의 전략 명작" 공개
- **시즌 랭킹 (3개월 주기)**:
  - 주간 토너먼트 누적 점수 → 시즌 등급 (브론즈→실버→골드→다이아→마스터)
  - 시즌 종료 보상: 등급별 한정 수호탑 스킨 + 마을 데코
- **타겟 페르소나**: P5 최동현 (경쟁), P3 박준혁 (효율 최적화)

### 3-4. 협동 모드 (확장 모드)

- **목적**: 소셜 플레이 경험 + 가족/친구 접점
- **로컬 협동**: 한 기기 화면 좌/우 분할, 공유 자원(골드), 양면 공격 맵
- **온라인 비동기 협동**: 플레이어 A 좌측 리플레이 → 코드 전송 → 플레이어 B 우측 배치 (A 고스트 재현)
- **협동 전용 스테이지**: 혼자서는 불가능한 양면 공격 맵 10종
- **승리 조건**: 두 플레이어 합산 점수
- **타겟 페르소나**: P4 정미경 (가족), P2 이서연 (친구)

### 3-5. 숏폼 체인 챌린지 (보조 모드)

- **목적**: 바이럴 유도 + 김하율 같은 유저에게 최소 접점
- **"마을 지키기 릴레이"**: A→B→C 체인 — 각자 1스테이지 클리어 후 패스
- **극한 제약 챌린지**: "타워 1개로 클리어", "골드 0 시작" 등
- **자동 숏폼 생성**: 10초 타임랩스 + 수채화 비주얼
- **타겟 페르소나**: P1 김하율 (숏폼), P5 최동현 (극한 도전)

### 3-6. 인터랙티브 스토리 (보조 모드)

- **목적**: 아트 게임 정체성 + 세계관 몰입
- **그림책 스타일 비네트**: 월드 간 스토리 장면, 플레이어 인터랙션(탭으로 꽃 피우기, 스와이프로 안개 걷기)
- **선택지 분기**: 선택에 따라 다음 월드 분위기/NPC 반응 변화
- **타겟 페르소나**: P6 한소율, P2 이서연

### 모드 간 계층 관계

```
[코어 계층] — 앱 첫 실행부터 접근
  ├── 스토리 퍼즐 모드 (전략 진행축)
  └── 마을 가꾸기 모드 (감성 진행축)

[확장 계층] — 스테이지 20+ 클리어 후 자연스럽게 노출
  ├── 비동기 PvP / 토너먼트
  └── 협동 모드 (로컬 + 온라인)

[보조 계층] — 특정 타겟/바이럴 목적
  ├── 숏폼 체인 챌린지
  └── 인터랙티브 스토리
```

## 4. 콘텐츠 시스템

### 프로시저럴 생성 알고리즘 설계

**맵 생성 파이프라인**:
```
1. 시드 결정 (스테이지 번호 = 시드값)
   │
2. 지형 레이아웃 생성 (Wave Function Collapse 기반)
   │ - 배치 가능 영역 / 경로 / 지형 효과(높은 지대, 물가)
   │
3. 경로 생성 (A* 기반, 1~3개 분기)
   │ - 시작점 → 마을 중심까지의 경로
   │ - 분기 수 = min(3, 1 + floor(스테이지 / 25))
   │
4. 잠의 파편 구성 결정
   │ - 파편 체력 = 기본값 x (1 + 스테이지 x 0.08)
   │ - 파편 종류 다양성 = min(스테이지 / 10, 최대 종류 수)
   │ - 잠의 물결 수 = 3 + floor(스테이지 / 15)
   │
5. 보스 배치 (매 10스테이지 — 프로시저럴 보스 패턴)
   │
6. 시뮬레이션 검증 (AI가 3종 전략으로 클리어 테스트)
   │
7. 합격 → 유저 제공 / 불합격 → 시드+1로 재생성
```

**잠의 파편 (적 유닛) 디자인 — 완전 탈전쟁화**:

| 파편 유형 | 시각 디자인 | 행동 특성 | 깨우기 방법 |
|-----------|-----------|-----------|------------|
| 안개 조각 | 뿌연 회색 구름 | 느리지만 넓게 퍼짐 | 빛의 탑으로 비추기 |
| 서리 결정 | 반투명 얼음 조각 | 빠르지만 약함 | 종소리로 깨뜨리기 |
| 그림자 웅덩이 | 보라빛 물웅덩이 | 느리고 체력 높음 | 여러 수호탑 집중 |

### 사운드 디자인 방향
- **BGM - 메인/마을**: 어쿠스틱 기타 + 피아노 + 현악 앙상블. 수채화 감성에 맞는 따뜻하고 잔잔한 루프 (Nils Frahm, Ólafur Arnalds 스타일)
- **BGM - 게임플레이**: 잔잔한 기반 위에 긴장감 추가 (리듬 퍼커션 레이어링, 물결 후반부로 갈수록 강화)
- **BGM - 보스/깊은잠**: 첼로/콘트라베이스 저음 추가, 어두운 분위기이나 공포는 아님
- **SFX - 수호탑 배치**: 부드러운 "뚝" + 작은 차임벨
- **SFX - 빛의 탑 작동**: 따뜻한 빛줄기 "슈웅~"
- **SFX - 종소리 탑 작동**: 맑은 종소리 (높낮이 랜덤 변화)
- **SFX - 잠 깨우기**: 크리스탈 깨지듯 맑은 "팅~" + 작은 바람 소리
- **SFX - 마을 깨어남**: 새 소리 + 물 흐르는 소리 + 풍경 효과음이 점점 추가
- **SFX - 마을 가꾸기**: 건물 배치 시 따뜻한 "둥" + 주민 환호 소리

### 애니메이션 원칙
- **수호탑 배치**: 씨앗에서 자라나듯 0.5초간 성장 애니메이션
- **잠의 파편 이동**: 떠다니듯 부드러운 움직임 (공격적 돌진 없음)
- **잠 깨우기 (파편 소멸)**: 빛 입자로 흩어지며 사라짐 (폭발/파괴 표현 없음)
- **마을 깨어남**: 안개가 서서히 걷히고, 색이 채워지고, 꽃이 피어나는 3단계 연출 (2~3초)
- **마을 데코**: 건물이 종이접기처럼 펼쳐지는 팝업북 스타일
- **UI 전환**: 수채화 물감이 번지듯 자연스러운 트랜지션
- **NPC 주민**: 2~3프레임의 귀여운 아이들 루프 (걷기, 손 흔들기, 앉기)

### 오프라인/서버리스 아키텍처

```
[클라이언트 (완전 로컬)]
    │
    ├── 게임 로직 (A* 패스파인딩, 웨이브 시스템, 수호탑 로직)
    ├── 프로시저럴 맵 생성 (시드 기반 결정론적)
    ├── 데이터 저장 (로컬 JSON + PlayerPrefs)
    ├── 리플레이 생성/재생 (입력 시퀀스 기반)
    ├── 수채화 아트워크 렌더링 (RenderTexture + 셰이더)
    └── 공유 (OS 네이티브 공유 시트 + 딥링크)

[선택적 네트워크]
    ├── 광고 SDK (AdMob)
    ├── IAP (스토어 네이티브)
    ├── Analytics (Firebase — 오프라인 캐싱)
    └── 마을 방문 (마을 코드 = 로컬 데이터 직렬화, P2P 교환)

[서버 불필요 영역]
    ├── 리더보드 → 리플레이 코드 교환으로 대체
    ├── 매칭 → 딥링크 직접 전송으로 대체
    └── 콘텐츠 업데이트 → 프로시저럴 무한 생성으로 대체
```

| 1 | **프로시저럴 맵 밸런스 붕괴** — 클리어 불가능 또는 너무 쉬운 맵 생성 | 높음 | 높음 | WFC 생성 규칙 + AI 3종 전략 사전 검증 필수. 클리어율 로컬 추적 → 적응형 난이도. 밸런싱 공식의 수학적 증명 |
| 2 | **정체성 딜레마** — "마을 가꾸기"와 "전략 퍼즐" 두 정체성 사이의 긴장 | 중 | 높음 | 두 모드를 독립 진행축으로 분리하되, "퍼즐 보상 → 마을 데코" 연결고리 유지. 어느 한 쪽에 집중하고 싶은 유저도 가능한 구조 설계. 온보딩에서 두 축을 명확히 안내 |
| 3 | **타워디펜스 장르 본질적 저항** — "적을 막는 구조" 자체에 대한 캐주얼 유저(P1/P4) 거부감 | 높음 | 중 | 탈전쟁화 완성 ("자연의 잠" 테마)이 최대 완화 장치. 마을 가꾸기를 독립 진행축으로 제공하여 퍼즐 비중 줄이기 가능. 그래도 안 되면 수용 가능한 한계로 인정 |
