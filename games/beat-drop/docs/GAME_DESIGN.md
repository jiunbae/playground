# Beat Drop (비트 드롭) — 심화 기획문서

---

## 1. 게임 비전

### 한줄 소개 (엘리베이터 피치)
**"내 폰에 저장된 음악이 자동으로 리듬 게임 스테이지가 되는, 원터치 뮤직 액션 게임"**

### 게임의 핵심 감정 (플레이어가 느껴야 할 것)
- **음악과의 일체감**: 비트에 완벽하게 맞춰 탭하는 순간, 음악과 내가 하나가 되는 몰입
- **드롭의 쾌감**: 곡의 클라이막스 구간에서 폭발하는 비주얼과 퍼펙트 콤보의 카타르시스
- **내 음악에 대한 애정**: "내가 좋아하는 이 노래"로 게임을 한다는 감정적 연결
- **경쟁의 짜릿함**: 같은 곡으로 친구와 점수를 겨루는 즉각적 경쟁의 자극

### 디자인 철학 (왜 이 게임이 존재하는가)
기존 리듬 게임은 "개발사가 준비한 곡"만 플레이할 수 있어 콘텐츠가 유한하고, 새 곡 추가에 라이선스 비용과 개발 비용이 필요하다. Beat Drop은 유저의 음악 라이브러리 전체를 실시간 분석하여 노트맵으로 변환함으로써, "내 노래가 곧 게임"이라는 무한 콘텐츠 구조를 실현한다. 서버 없이, 라이선스 없이, 유저가 가진 모든 음악이 스테이지가 되는 자급자족 생태계. 여기에 원터치 조작과 음악 반응형 비주얼이 결합되어, 리듬 게임 초보부터 하드코어까지 모두를 아우른다.

### 타겟 유저 프로필

| 구분 | 페르소나 | 핵심 욕구 | 기대 만족도 |
|------|----------|-----------|-------------|
| **1차 타겟** | P1 김하율 (18세, 고등학생) | 틱톡 숏폼 콘텐츠, 친구 대결, 즉각적 재미 | 매우 높음 (43/50) |
| **1차 타겟** | P5 최동현 (22세, 대학생 남성) | 고난이도 도전, 랭킹 경쟁, 노트맵 제작 | 매우 높음 (42/50) |
| **2차 타겟** | P6 한소율 (34세, 프리랜서 여성) | 비주얼 캔버스, 인디 음악 시각화, 리듬 스토리 | 매우 높음 (42/50) |
| **2차 타겟** | P2 이서연 (26세, 직장인 여성) | 힐링 플레이, 감성 리캡, 커플 듀엣 | 높음 (41/50) |
| **2차 타겟** | P3 박준혁 (32세, 하드코어 게이머) | 노트맵 역분석, 레이팅 시스템, 시스템 깊이 | 보통 (33/50) |
| **3차 타겟** | P4 정미경 (43세, 주부) | 가이드 비주얼로 리듬게임 입문, 자녀와 듀엣 | 낮음 (23/50) |
| **비타겟** | 음악 스트리밍 전용 유저 (로컬 음원 미보유) | 핵심 USP "내 음악으로 플레이" 작동 불가 | - |

## 2. 코어 메커니즘 상세

### 핵심 동사 (플레이어가 하는 행동)
1. **탭한다 (Tap)** — 비트에 맞춰 화면을 터치하여 노트를 처리한다
2. **홀드한다 (Hold)** — 긴 노트에서 손가락을 누른 채 유지한다
3. **스와이프한다 (Swipe)** — 방향 노트에 맞춰 좌/우/상/하로 스와이프한다
4. **감상한다 (Appreciate)** — 음악 반응형 비주얼과 리듬 스토리를 감상한다
5. **경쟁한다 (Compete)** — 실시간 대결, 플레이리스트 배틀로 점수를 겨룬다

### 조작 방식 (터치 제스처별 매핑)

| 제스처 | 동작 | 난이도 | 상세 |
|--------|------|--------|------|
| **탭 (Tap)** | 기본 노트 처리 | 전체 | 화면 어디든 탭. 타이밍에 따라 Perfect/Great/Good/Miss 판정 |
| **롱프레스 (Hold)** | 긴 노트 처리 | Normal+ | 노트 시작점에서 탭 후 끝점까지 누른 채 유지. 도중 떼면 Miss |
| **스와이프** | 방향 노트 | Hard+ | 좌/우/상/하 방향으로 스와이프. 한 손 전용 모드에서는 비활성화 |
| **멀티탭 (동시 2점)** | 동시 노트 | Expert | 양손으로 동시 2개 노트 처리 |
| **가이드 따라 탭** | 가이드 비주얼 | Zen/Easy | 화면 중앙의 수축 원에 맞춰 탭. 별도 제스처 불필요 |

**한 손 전용 모드**: 스와이프 제거, 탭+롱프레스만. 노트가 화면 하단 중앙에 집중 배치

### 난이도 곡선 설계

```
난이도      노트 밀도    입력 유형          판정창    라이프    대상 유저
─────────────────────────────────────────────────────────────────
Zen         메인비트만   탭 only           매우 넓음  없음     P4, 신규
Easy        메인비트     탭 only           넓음      없음     P4, P2
Normal      비트+멜로디  탭+홀드            보통      있음     P2, P6
Hard        전 요소      탭+홀드+스와이프    좁음      있음     P1, P5
Expert      전 요소+서브  탭+홀드+스와이프+멀티 매우 좁음 있음   P5, P3
```

**드롭 구간 특별 처리**:
- 모든 난이도에서 드롭 진입 시 노트 밀도 1.5~2배 증가
- 비주얼 폭발적 변화 (배경 색조 전환, 파티클 폭발)
- 드롭 구간 퍼펙트 클리어 시 점수 x2 보너스

### 세션 설계

| 구분 | 시간 | 시나리오 | 활동 |
|------|------|----------|------|
| **최소 세션** | 30초 | 지하철 한 정거장, 신호 대기 | 드롭 구간만 30초 클리어 |
| **최적 세션** | 3~5분 | 출퇴근 지하철 이어폰 착용 시 | 좋아하는 곡 1곡 풀 플레이 |
| **확장 세션** | 10~15분 | 점심 휴식, 카페 | 3~5곡 연속 또는 리듬 스토리 1세트 |
| **몰입 세션** | 30분+ | 주말 여유 시간 | 실시간 대결 연속, 노트맵 에디팅, 플레이리스트 배틀 |

## 3. 게임 모드별 상세

### 모드 계층 구조

```
[핵심 모드]
├── 솔로 플레이 (곡 선택 → 리듬 플레이) ★ — 게임의 정체성
│
[서브 모드]
├── 실시간 1v1 대결 — 같은 곡 동시 점수 경쟁
├── 듀엣 모드 — 저음/고음 분담 협동
├── 플레이리스트 배틀 — 3곡 vs 3곡 취향 대결
├── 커스텀 노트맵 에디터 — UGC 구간 제작
├── 비주얼 캔버스 모드 — 음악 시각화 도구
├── 리듬 스토리 — 연속 플레이 내러티브
│
[보너스 모드]
├── 주간 챌린지곡 — 매주 동일 시드
├── Zen 모드 — 실패 없는 힐링 리듬
└── 내 음악 스토리 — 연간 리캡
```

### 솔로 플레이 (핵심 모드)
- **목적**: 내 음악으로 리듬 게임을 즐기는 핵심 경험
- **규칙**: 곡 선택 → 오디오 실시간 분석 → 자동 노트맵 생성 → 리듬 플레이. 난이도/세션 길이/조작 방식 유저 선택
- **승리 조건**: 곡 끝까지 라이프 유지(Hard/Expert), 등급 S~D 부여, 최고 점수/콤보 갱신
- **타겟 페르소나**: 전체 (P1~P6)

### 실시간 1v1 대결 (서브 모드)
- **목적**: 같은 곡으로 동시 점수 경쟁. 소셜 자극
- **규칙**: 같은 곡/난이도를 동시 플레이. P2P 방식으로 판정은 로컬, 점수만 동기화. 상대 콤보/판정을 상단 미니 게이지로 실시간 표시. 드롭 구간 x2 배율로 역전 가능
- **승리 조건**: 곡 종료 시 총점 비교, 높은 쪽 승리
- **매칭**: 친구 초대(링크 공유) 또는 랜덤 매칭
- **타겟 페르소나**: P1 김하율, P5 최동현

### 듀엣 모드 (서브 모드)
- **목적**: 실력 차이 극복, 커플/가족 소셜 경험, 바이럴 소재
- **규칙**: 같은 곡에서 2인이 각각 저음부/고음부(또는 좌/우 채널) 노트를 분담. 한 사람이 놓쳐도 파트너가 커버 가능한 "구원 시스템". 결과 화면에 "우리의 호흡: XX%", 파트별 기여도, 하이라이트 구간 표시
- **승리 조건**: 2인 합산 점수 + 협동 정확도(동시 Perfect 비율) 기반 등급
- **플레이 방식**: 오프라인(같은 기기, 화면 분할) + 온라인(P2P)
- **타겟 페르소나**: P4 정미경(자녀와), P2 이서연(커플), P1 김하율(친구)

### 플레이리스트 배틀 (서브 모드)
- **목적**: 음악 취향이 경쟁 변수가 되는 독특한 대결
- **규칙**: 각자 3곡 플레이리스트 지정 → 총 6곡을 번갈아 플레이 → 합산 점수. 상대의 곡을 처음 플레이하는 "어웨이 매치" 긴장감
- **승리 조건**: 6곡 총합 점수 비교
- **타겟 페르소나**: P1 김하율, P5 최동현

### 커스텀 노트맵 에디터 (서브 모드)
- **목적**: UGC 생태계, 코어 유저 시스템 깊이 확장
- **규칙**: 자동 생성 노트맵의 특정 구간(최대 30초)을 터치 기반으로 수정. 노트 추가/삭제/이동. "내가 만든 구간"을 링크로 공유하여 친구에게 도전 유도
- **레이팅**: 난이도 자동 측정(노트 밀도 x BPM 대비 복잡도) + 유저 평가(좋아요/별점) → 품질 등급 S/A/B/C
- **명예의 전당**: S등급 노트맵 등재, 제작자에게 "노트맵 아키텍트" 칭호
- **타겟 페르소나**: P3 박준혁, P5 최동현

### 비주얼 캔버스 모드 (서브 모드)
- **목적**: 리듬 게임을 넘어선 음악 시각화 도구
- **규칙**: 리듬 게임 UI 완전 제거. 음악의 에너지/주파수/비트에 반응하는 순수 비주얼 경험. 비주얼 프리셋 선택 가능
- **내보내기**: 라이브 월페이퍼(Android) / 스크린세이버 / 15초 클립 영상
- **타겟 페르소나**: P6 한소율, P2 이서연

### 리듬 스토리 (서브 모드)
- **목적**: 감성적 세션, 음악으로 하루를 이야기
- **규칙**: 3~5곡을 연속 플레이. 비주얼이 내러티브로 전개(일출→정오→석양→밤하늘). 곡의 에너지/분위기에 따라 배경이 유기적 변화. 매주 추천 플레이리스트(3곡 세트)를 날짜 시드로 자동 제공
- **완주 보상**: "나의 오늘 하루" 비주얼 리캡 카드 자동 생성 (30초 타임랩스 + 인스타 스토리 최적화 포맷)
- **타겟 페르소나**: P2 이서연, P6 한소율

### Zen 모드 (보너스 모드)
- **목적**: 리듬감 부족 유저 진입 경로, 힐링
- **규칙**: 실패 판정 없음, BPM 0.7x 감속, 시각 효과 축소. 가이드 비주얼(수축 원형 펄스) ON/OFF 선택
- **타겟 페르소나**: P4 정미경

### 주간 챌린지곡 (보너스 모드)
- **목적**: 매주 접속 동기, 글로벌 비교
- **규칙**: 날짜 시드로 내장 샘플곡 1곡 자동 선정. 전 세계 동일 조건 경쟁
- **타겟 페르소나**: P1, P5 (리더보드)

## 4. 콘텐츠 시스템

### 프로시저럴 생성 알고리즘 설계

#### 오디오 분석 → 노트맵 자동 생성 파이프라인

```
입력: 로컬 음원 파일 (MP3/AAC/WAV 등)
    │
    ▼
[1단계] 오디오 전처리
- 샘플레이트 정규화 (44.1kHz)
- 스테레오 → 모노 다운믹스 (분석용)
- 듀엣 모드용 주파수 분리 (저음부/고음부)
    │
    ▼
[2단계] BPM 감지 & 비트 추적
- Onset Detection: 에너지 변화량의 피크 검출
- Beat Tracking: 자기상관(Autocorrelation) 기반 BPM 추정
- 다운비트 감지: 마디의 첫 박 위치 결정
- 결과: BPM + 비트 타임스탬프 배열
    │
    ▼
[3단계] 에너지 & 스펙트럼 분석
- Spectral Flux: 주파수 스펙트럼 변화율 (노트 밀도 결정)
- 에너지 레벨: RMS 기반 구간별 에너지 (드롭 감지)
- 주파수 대역 분리: 저음(~250Hz), 중음(250~2kHz), 고음(2kHz~)
- 드롭 감지: 에너지 급상승 구간 자동 마킹
    │
    ▼
[4단계] 노트맵 생성
- 난이도별 필터 적용:
  · Easy: 메인 비트만 (1박 단위)
  · Normal: 비트 + 강세 멜로디 (2박 단위)
  · Hard: 전 비트 + 멜로디 + 서브비트 (1/2박)
  · Expert: 모든 음악적 이벤트 (1/4박까지)
- 노트 유형 할당:
  · 짧은 이벤트 → Tap
  · 지속 이벤트(>0.5초) → Hold
  · 에너지 변화 방향 → Swipe 방향
  · 동시 이벤트 → Multi-tap (Expert)
- 노트 위치 배치: 화면을 3~5레인으로 분할, 인접 노트 간 물리적 거리 최적화
    │
    ▼
[5단계] 비주얼 매칭
- BPM → 배경 펄스 주기 결정
- 에너지 레벨 → 파티클 밀도/크기
- 주파수 대역 → 색조 매핑 (저음:보라/파랑, 고음:주황/빨강)
- 드롭 구간 → 전체 화면 비주얼 전환 트리거
    │
    ▼
출력: 노트맵 (JSON) + 비주얼 파라미터 맵 + 드롭 구간 마커
```

### 공유 이미지/카드 디자인 명세

#### 결과 카드
- **크기**: 1080x1080px
- **구성**: 비주얼 테마 배경(블러) + 중앙 원형 앨범아트(유저 곡의 메타데이터에서 추출, 없으면 자동 생성 비주얼) + 하단 바(곡명, 아티스트, 점수, 등급, 최대 콤보) + "Beat Drop" 워터마크
- **등급별 연출**: S등급 = 금색 테두리 + 파티클, A = 은색, B = 동색, C/D = 기본

#### 하이라이트 클립
- **길이**: 15초
- **구성**: 드롭 구간 시작 3초 전부터 시작 → 드롭 진입 비주얼 폭발 → 콤보 카운트 오버레이 → 엔드카드(점수/등급)
- **음악 포함**: 해당 구간 원곡 음원 + 게임 SFX 믹스

#### 듀엣 "우리의 호흡" 카드
- **크기**: 1080x1350px (인스타 스토리)
- **구성**: 상단(2인 프로필 나란히) + 중앙("우리의 호흡: 94%" 대형 숫자 + 파트별 기여도 바) + 하단(곡명 + 하이라이트 구간 표시) + 배경(비주얼 테마 블러)

### 핵심 UI 요소 명세

#### 메인 화면
- **상단**: "나의 음악 여정" 위젯 (총 플레이 곡 수, 이번 주 하이라이트) — 핵심 USP 가시성
- **중앙**: "플레이 시작" 대형 CTA + 최근 플레이 곡 3개 썸네일
- **하단 탭바**: 홈 / 플레이 / 대결 / 뮤직 비주얼 / 프로필

#### 리듬 플레이 화면
- **노트 레인**: 화면 상단에서 하단으로 노트 낙하 (3~5레인, 난이도별)
- **판정 라인**: 하단 20% 위치, 노트가 여기 도달할 때 탭
- **판정 이펙트**: Perfect(금색 파티클), Great(은색), Good(동색), Miss(적색 흔들림)
- **콤보 카운터**: 우상단, 콤보 수치 + 배수 표시
- **라이프 게이지**: 좌상단 (Hard/Expert만), Miss 시 감소
- **상대 게이지** (대결 시): 상단 바, 상대 점수/콤보 미니 표시
- **가이드 비주얼** (Zen/Easy): 화면 중앙 수축 동심원 펄스

#### 결과 화면
- **등급 대형 표시**: 화면 중앙 등급 문자 + 파티클 (S등급 시 금색 폭발)
- **통계**: 점수, Perfect/Great/Good/Miss 비율, 최대 콤보, 정확도 %
- **하이라이트 미리보기**: 15초 클립 자동 재생 (음소거 가능)
- **액션 버튼**: [공유] [재도전] [다음 곡] [대결 보내기]

### 사운드 디자인 방향

#### BGM
- **게임 자체에 BGM 없음** — 유저의 음악이 BGM
- **메뉴/UI**: 미니멀 앰비언트 루프 (15초, 매우 조용함). 곡 선택 화면에서는 선택된 곡의 5초 프리뷰 자동 재생
- **비주얼 캔버스 모드**: 유저 음악이 풀볼륨 BGM

#### SFX
- **노트 히트 (Perfect)**: 짧고 선명한 "틱" + 미세 진동. 음높이가 콤보에 따라 점차 상승
- **노트 히트 (Great)**: 약간 뭉툭한 "톡"
- **노트 히트 (Good)**: 저음 "뚝"
- **Miss**: 부드러운 "쉬..." (불쾌하지 않되, 실수 인지 가능)
- **콤보 x10 달성**: 짧은 상승 아르페지오
- **드롭 진입**: 저음 럼블 + 고음 라이저 (영화 트레일러 스타일 빌드업)
- **S등급 달성**: 팡파레 + 심벌 크래시 (화려하되 과하지 않게, 1.5초)
- **롱노트 홀드 중**: 저음 지속음 (음악과 간섭하지 않는 주파수 대역)

**핵심 원칙**: 모든 SFX는 유저의 음악과 간섭하지 않도록 주파수 대역 분리. SFX 볼륨은 음악의 30~50% 수준. 설정에서 SFX/음악 볼륨 독립 조절 가능

#### 앰비언트
- 메뉴 화면: 매우 가벼운 앰비언트 드론 (공간감)
- 게임 중: 없음 (유저 음악만)

### 애니메이션 원칙

1. **비트 동기화**: 모든 UI 요소의 펄스/스케일 변화는 감지된 BPM에 동기화
2. **에너지 반응**: 곡의 에너지 레벨에 비례하여 파티클 밀도, 배경 밝기, 색 채도 자동 변화
3. **드롭 폭발**: 드롭 진입 순간 300ms 동안 화면 전체 플래시 + 파티클 폭발 + 색조 전환
4. **판정 피드백**: 탭 후 50ms 이내 시각 피드백(파티클 + 판정 텍스트) — 즉각적 반응감
5. **콤보 연출**: 콤보 수가 10배수 달성 시마다 카운터 스케일업(1.5x → 1x 바운스, 200ms)
6. **프레임 예산**: 게임 플레이 중 60fps 절대 유지. 비주얼 이펙트가 프레임 드롭 유발 시 파티클 수 자동 감소

### 대응 방안 총괄 원칙
1. **USP 수호**: "내 음악이 곧 게임"이라는 핵심 가치를 어떤 리스크 대응에서도 훼손하지 않는다
2. **오디오 품질 최우선**: 리듬 게임의 생명은 판정 정확도. 레이턴시/분석 품질은 출시 전 완벽히 해결
3. **오프라인 폴백 필수**: 온라인 기능이 실패해도 솔로 플레이는 100% 정상 동작
4. **로컬 음원 미보유 대비**: 내장 샘플곡만으로도 핵심 경험 가능하도록 설계. 스트리밍 파트너십은 장기 로드맵
5. **단계적 기능 개방**: 출시 시 솔로+대결 핵심, 검증 후 에디터/갤러리/배틀 점진적 활성화
