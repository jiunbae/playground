# Roguelike Deckbuilder (카드의 탑) - 심화 기획문서

---

## 1. 게임 비전

### 한줄 소개 (엘리베이터 피치)
> "카드 한 장의 선택이 운명을 바꾼다 — 매 런마다 새로운 시너지를 발견하는 모바일 네이티브 로그라이크 덱빌더."

### 게임의 핵심 감정
- **발견의 쾌감**: "이 카드와 저 카드를 합치면 이런 효과가?" — 시너지 발견의 유레카 순간
- **전략적 성취감**: 제한된 자원 내에서 최적의 빌드를 완성했을 때의 지적 만족
- **극적 긴장감**: "HP 1 남았는데 이 카드 한 장으로 역전할 수 있을까?" — 로그라이크 특유의 긴장
- **숙련의 자부심**: Ascension 20 클리어, 카드 마스터리 레전더리 달성 — "나는 이 게임을 정복했다"
- **공유의 즐거움**: 독창적 빌드를 레시피 카드로 만들어 커뮤니티에 자랑하는 기쁨

### 디자인 철학
덱빌더 로그라이크는 본질적으로 "제약 속 최적화"의 게임이다. 매 런마다 랜덤하게 주어지는 카드와 유물 중에서 최선의 조합을 찾아내는 과정 자체가 게임의 핵심이다. 카드의 탑은 이 "발견"을 극대화하기 위해 속성 간 크로스 시너지 시스템을 도입하여, 단순한 "같은 속성 모으기"를 넘어 "다른 속성 간 창발적 조합"이라는 독자적 재미를 제공한다. 모바일 네이티브로 처음부터 설계하여, PC 포팅 게임이 가진 UX 한계를 뛰어넘고, 5분 퀵 런부터 40분 풀 런까지 유연한 세션을 지원한다. 범용성보다 깊이를 택한 니치 전략 — "모든 사람이 좋아하는 게임"이 아닌 "좋아하는 사람에게는 최고인 게임"을 만든다.

### 타겟 유저 프로필

**1차 타겟: 코어 덱빌더 유저 (전체 유저의 25~30%)**
- 18~35세, Slay the Spire/Monster Train 경험자
- 시스템을 깊이 파고들어 최적화하는 것을 즐기는 전략가
- 수백 시간 리플레이할 의지가 있는 유저
- 높은 과금 의향 (ARPPU 높음)
- 대표 페르소나: P3 박준혁(32세, 백엔드 개발자), P5 최동현(22세, 대학생)

**2차 타겟: 미드코어/내러티브 유저 (전체 유저의 20~25%)**
- 20~40세, 카드게임에 관심은 있으나 StS급 하드코어까지는 아닌 유저
- 아트 스타일이나 스토리에도 매력을 느끼는 유저
- 캐주얼 난이도 + 스토리로 진입, 점차 깊이에 빠지는 유형
- 대표 페르소나: P6 한소율(34세, 일러스트레이터)

**3차 타겟: 스트리머 시청자층 (전체 유저의 15~20%)**
- 직접 플레이보다 스트리머/유튜버의 플레이를 먼저 접하는 유저
- 스트리머 모드의 관전자 투표/방해를 통해 간접 참여 후 직접 설치로 전환
- 대표 페르소나: P1 김하율(18세, 고등학생 — 시청자로서)

**비타겟 (전략적 포기)**
- P4 정미경(43세, 전업주부) 유형: 카드게임 특유의 복잡성이 근본적 장벽. 캐주얼화하면 핵심 매력 훼손
- 단순 매치3, 타이쿤 등 캐주얼 퍼즐 유저
- 덱빌더/로그라이크에 관심 없는 일반 캐주얼 유저

## 2. 코어 메커니즘 상세

### 핵심 동사 (플레이어가 하는 행동)
1. **선택한다 (Choose)**: 카드 보상, 경로 분기, 이벤트 선택지 — 매 순간이 의미 있는 선택
2. **조합한다 (Combine)**: 카드 간 시너지를 발견하고 빌드를 구축
3. **플레이한다 (Play)**: 턴제 전투에서 에너지를 관리하며 카드를 사용
4. **적응한다 (Adapt)**: 랜덤하게 주어지는 카드/유물에 맞춰 빌드 전략을 수정
5. **공유한다 (Share)**: 빌드 레시피 카드를 커뮤니티에 공유하고 토론

### 조작 방식 (터치 제스처별 매핑)

| 제스처 | 전투 중 | 노드 맵 | 공통 |
|--------|--------|--------|------|
| 탭 | 카드 선택/확인 | 노드 선택 | 버튼 선택 |
| 드래그 | 카드를 적/아군에게 드래그 → 사용 | 맵 스크롤 | 리스트 스크롤 |
| 스와이프 위 | 핸드에서 카드를 위로 스와이프 → 사용 | - | - |
| 스와이프 좌우 | 핸드 카드 넘겨보기 | - | 화면 전환 |
| 길게 누르기 | 카드 상세 정보 + 관련 시너지 표시 | 노드 정보 팝업 | 도움말 |
| 더블 탭 | 카드 즉시 사용 (빠른 플레이) | - | - |
| 핀치 줌 | 전체 핸드 보기 ↔ 확대 | 맵 줌 인/아웃 | - |

## 3. 게임 모드별 상세

### 스탠다드 런 (메인 경험)

**목적**: 3 Act 던전을 공략하며 시너지 빌드를 완성하는 핵심 덱빌딩 경험
**규칙**:
- 캐릭터 선택 → 시작 덱 + 고유 능력으로 시작
- Act 1(기반 구축) → Act 2(시너지 완성) → Act 3(최종 도전)
- 노드 맵에서 경로 선택: 전투/엘리트/상점/이벤트/휴식/보물
- 카드 획득/제거/업그레이드로 덱 빌딩
- 사망 시 런 종료 (로그라이크)
**승리 조건**: Act 3 최종 보스 격파
**난이도**: 캐주얼 / 노멀 / Ascension 1~20
**타겟 페르소나**: P3 박준혁(핵심), P5 최동현, P6 한소율(내러티브 포함)

### 퀵 런 (숏 세션)

**목적**: 5~10분 내 완결되는 압축 덱빌딩 경험, 모바일 환경 최적화
**규칙**:
- 1 Act 압축 구성 (전투 3~4회 + 보스 1회)
- 테마별 프리셋 덱으로 시작 (화염 공격덱, 냉기 방어덱 등)
- 프리셋 덱 제한 조건 하 "최적 플레이" 추구
- 3x 배속 기본
**승리 조건**: 보스 격파 + 점수 최대화
**경쟁 시스템**: 일일 퀵 런 시드(동일 조건 비교), 프리셋별 리더보드, 주간 시즌제
**타겟 페르소나**: P5 최동현(경쟁), P3 박준혁(최적화), P1 김하율(짧은 세션)

### 일일 도전

**목적**: 매일 동일 조건의 런으로 전 유저 간 비교/공유
**규칙**:
- 날짜 시드 기반 고정 조건 런 (예: "화염 카드만", "시작 HP 1", "유물 10개 시작")
- 모든 유저 동일 맵/카드/이벤트
- 1일 1회 제한
**승리 조건**: 클리어 + 점수 기반 순위
**타겟 페르소나**: P3 박준혁, P5 최동현

### 주간 빌드 챌린지

**목적**: 특수 제약 조건으로 창의적 빌딩 유도 + 커뮤니티 토론 촉진
**규칙**:
- 매주 월요일 특수 조건 런 제시 (예: "냉기 카드만", "유물 0개", "카드 15장 이하 덱", "1턴 킬 보스")
- 챌린지 기간: 월~일 (7일)
- 챌린지 클리어 시 한정 카드 스킨 보상 (해당 주에만 획득 가능)
- 챌린지별 전용 리더보드
**승리 조건**: 챌린지 조건 충족 + 클리어
**타겟 페르소나**: P3 박준혁(최적화 퍼즐), P5 최동현(한정 보상)

### 스트리머 모드

**목적**: 스트리머/유튜버의 콘텐츠 제작 최적화, 관전자 참여형 플레이
**규칙**:
- 트위치/유튜브 채팅 연동
- 카드 보상 선택 시 관전자 실시간 투표 (10초 투표 시간)
- "관전자 방해" 옵션: 시청자 투표로 추가 제약 (에너지 -1, 랜덤 카드 삭제 등)
- 스트리머 전용 오버레이: 현재 덱/시너지/HP 실시간 표시
**승리 조건**: 스탠다드 런과 동일
**타겟 페르소나**: P5 최동현(미래 스트리머), P1 김하율(시청자)

### 아트 갤러리

**목적**: 카드 아트의 감상적 가치를 독립 콘텐츠로 승격
**규칙**:
- 수집한 카드의 풀 일러스트를 갤러리 형태로 감상
- 마스터리 등급별 해금: 노멀(기본 아트) → 레어(설정 텍스트) → 에픽(개발자 코멘터리) → 레전더리(풀 일러스트)
- NPC 캐릭터별 배경 스토리/설정 텍스트 열람
**타겟 페르소나**: P6 한소율(아트 감상), P2 이서연(비주얼)

## 4. 콘텐츠 시스템

### 프로시저럴 생성 알고리즘 설계

#### 카드 밸런스 시스템

**카드 데이터 구조**:
```json
{
  "id": "fire_strike_01",
  "name": "화염 참격",
  "element": "fire",
  "role": "attack",
  "rarity": "common",
  "energy_cost": 1,
  "base_damage": 8,
  "upgraded_damage": 12,
  "effects": ["burn_1"],
  "synergy_tags": ["fire_dot", "fire_basic"],
  "description": "적에게 8 데미지를 주고 화상 1을 부여한다.",
  "mastery": {
    "uses": 0,
    "win_rate": 0.0,
    "max_single_damage": 0,
    "grade": "normal"
  }
}
```

**카드 밸런스 공식**:
```
카드 파워 점수 = (기본 효과값 × 효과 계수) / 에너지 비용
  + 시너지 잠재력 보너스 (0.1~0.5)
  - 조건부 패널티 (리스크 카드의 경우)

목표: 모든 카드의 "상황 최적 파워 점수"가 2.0~3.0 범위
  * 코먼: 기본 파워 2.0~2.5
  * 언코먼: 기본 파워 2.5~3.0 (조건부)
  * 레어: 기본 파워 3.0+ (시너지 의존적)
```

**시너지 밸런스**:
```
속성 시너지 (15개):
  3장 보유: 속성 보너스 Lv.1 (약한 패시브)
  5장 보유: 속성 보너스 Lv.2 (강한 패시브)
  7장 보유: 속성 보너스 Lv.3 (강력한 패시브)

크로스 시너지 (30개+):
  2속성 조합: 10개 (5C2)
  3속성 조합: 10개 (5C3) — 히든 시너지
  4속성 조합: 5개 (5C4) — 울트라 레어 시너지
  5속성 조합: 1개 — 궁극 시너지

시너지 밸런스 원칙:
  단일 속성 특화 빌드 ≈ 크로스 시너지 빌드 ≈ 유물 기반 빌드
  (어떤 빌드 스타일도 Ascension 20 클리어 가능해야 함)
```

#### 던전 생성 알고리즘

**노드 맵 생성**:
```
입력: 시드(seed), Act 번호, 난이도
출력: 노드 그래프 (15~18개 노드/Act)

알고리즘:
1. 시작 노드 생성
2. 층별 노드 수 결정 (2~4개/층, 정규분포)
3. 노드 타입 할당 (비율 파라미터):
   Act 1: 전투 55%, 이벤트 15%, 상점 10%, 휴식 10%, 엘리트 5%, 보물 5%
   Act 2: 전투 45%, 이벤트 15%, 상점 10%, 휴식 10%, 엘리트 15%, 보물 5%
   Act 3: 전투 40%, 이벤트 10%, 상점 10%, 휴식 10%, 엘리트 20%, 보물 10%
4. 노드 간 연결 생성 (분기 2~3, 합류점 포함)
5. 규칙 검증:
   - 엘리트 연속 배치 금지
   - 보스 직전 반드시 휴식 또는 상점 배치
   - 최소 1개 경로는 엘리트 우회 가능
6. 보스 노드 추가 (Act 끝)
```

**적 AI 행동 패턴**:
```
적 행동 = 상태 기반 확률 테이블

예: "화염 골렘" (Act 1 일반)
  HP > 50%: 공격(60%), 강화(30%), 방어(10%)
  HP ≤ 50%: 공격(40%), 자폭 충전(40%), 방어(20%)
  자폭 충전 후: 자폭(100%) — 3턴 예고

보스 AI: 페이즈 전환 시스템
  Phase 1 (HP 100~60%): 기본 패턴 루프 (3턴 주기)
  Phase 2 (HP 60~30%): 강화 패턴 + 소환
  Phase 3 (HP 30~0%): 광폭화 (공격력 2배, 방어 해제)
```

**카드 보상 알고리즘**:
```
카드 보상 풀 생성:
1. 기본 풀: 해금된 전체 카드에서 레어리티 가중 랜덤
   코먼 60%, 언코먼 30%, 레어 10%
2. 시너지 보정: 현재 덱의 주력 속성 카드 +20% 가중치
3. 크로스 시너지 유도: 현재 덱에서 1장만 추가하면 발동하는 크로스 시너지 카드 +30% 가중치
4. 다양성 보정: 최근 3회 보상에서 나온 카드 -50% 가중치
5. 3장 중 선택 (+ 스킵 옵션)
```

|------|--------|--------|--------|--------|------|
| 출시 시 | 200장+ (5속성×4역할) | 50개+ | 3명 | 속성 15 + 크로스 30+ | Act 3 + 이벤트 25개 |
| 3개월 | +40장 (카드팩 1) | +10개 | +1명 | +5 크로스 시너지 | 시즌 1 배틀패스 |
| 6개월 | +40장 (카드팩 2) | +10개 | +1명 | +5 크로스 시너지 | 스트리머 모드 정식 |
| 1년 | 총 320장+ | 80개+ | 5명 | 총 60+ 시너지 | 시즌 12회, 챌린지 52회 |

### 시즌/이벤트 설계

**시즌 배틀 패스 (4주 단위)**:
- 무료 트랙 (30단계): 골드, 카드팩, 기본 카드 스킨
- 프리미엄 트랙 (30단계): 한정 카드 스킨, 애니메이션 프레임, 한정 칭호, 프리미엄 레시피 카드 프레임
- 경험치 소스: 일일 도전 클리어, 주간 챌린지 클리어, 스탠다드 런 클리어, 퀵 런 클리어

**주간 빌드 챌린지 (매주 월요일 갱신)**:

  - "수집가": 카드 40장 이상 덱으로 클리어
  - "속성 마스터": 5속성 카드를 모두 포함한 덱으로 클리어

| 카드 마스터 | 전체 200장 레전더리 | "카드 마스터" 궁극 칭호 + 전용 카드 뒷면 | 300~500시간 |
| 시너지 100% | 전체 시너지 발견 | "시너지의 주인" 궁극 칭호 | 200시간 |

### 수집 요소

1. **카드 도감**: 전체 카드 목록 + 마스터리 등급 표시. 레전더리 달성 시 풀 일러스트 + 애니메이션 프레임 해금
2. **시너지 도감**: 발견한 시너지 조합 목록 + 발동 조건 설명. 히든 시너지는 "???" 표시 → 발견 시 해금
3. **유물 도감**: 전체 유물 + 획득 기록 + 유물 간 시너지 조합
4. **빌드 히스토리**: 과거 런의 전체 빌드(덱/유물/시너지) 기록. 성공/실패 태그, 점수 기록
5. **카드 마스터리**: 카드별 사용 횟수/승률/최대 데미지 추적 → 노멀→레어→에픽→레전더리 아트 진화
6. **NPC 관계도**: 반복 런에서 NPC와의 대화/선택 누적 → 새로운 대화/이벤트 해금

### 업적/도전과제 시스템

**전투 업적**:
- "유리대포": HP 1인 상태에서 보스 격파
- "999 데미지": 단일 카드로 999 이상 데미지
- "무피격 보스": 보스전에서 데미지 0으로 클리어
- "1턴 킬": 보스를 1턴에 격파
- "카드 1장 덱": 카드 1장만으로 Act 3 클리어

**빌드 업적**:
- "속성 전문가": 단일 속성 카드만으로 클리어 (5속성 각각)
- "무지개 빌더": 5속성 카드를 모두 포함한 덱으로 클리어
- "미니멀리스트": 카드 10장 이하 덱으로 클리어
- "수집광": 카드 50장 이상 덱으로 클리어
- "유물 부자": 유물 15개 이상으로 클리어

**진행 업적**:
- "시너지 헌터": 1런에서 5개 이상 시너지 동시 발동
- "스피드러너": 스탠다드 런 15분 이내 클리어
- "철인": Ascension 20 모든 캐릭터로 클리어
- "주간 챌린저": 연속 4주 챌린지 클리어

**숨겨진 업적**:
- "카오스 빌드": 아무 시너지 없이 Ascension 10 클리어
- "상인의 친구": 상점에서 50,000 골드 이상 소비 (누적)
- "NPC 마스터": 모든 NPC의 숨겨진 대화 이벤트 발견

```
[유입 경로 1: 스트리머]

    ├─ 스트리머가 관전자 투표 모드로 플레이
    ├─ 시청자가 "이 게임 재밌겠다" → 설치
    └─ 직접 플레이 → 레시피 카드 공유 → 커뮤니티 확산

[유입 경로 2: 커뮤니티]
    │
    ├─ 레딧/디스코드에서 빌드 레시피 카드 발견
    ├─ "이 빌드 도전해볼래" → 설치 → 빌드 코드 입력
    └─ 자신만의 빌드 발견 → 레시피 카드 공유 → 순환

    │
    ├─ 인스타/트위터에서 카드 아트 또는 업적 카드 발견
    ├─ "아트 예쁘다" / "Ascension 20을?" → 설치
    └─ 플레이 → 갤러리/업적 공유 → 순환
```

### 친구/대결 시스템 상세

카드의 탑은 PvP 대전이 아닌 **PvE 솔로 + 간접 경쟁** 모델:

**일일 도전 비교**: 같은 시드 런의 결과(점수/턴 수/HP)를 친구와 비교
**퀵 런 리더보드**: 주간 시드 기반, 같은 조건에서 점수 경쟁
**빌드 코드 도전**: 친구의 레시피 카드에 포함된 빌드 코드를 입력하면 동일 시드로 도전 → 결과 비교
**주간 챌린지 리더보드**: 같은 챌린지 조건에서 점수/속도 경쟁

### 애니메이션 원칙

1. **카드 모션**: 드래그 시 카드가 살짝 기울어지며 따라옴. 사용 시 속성별 이펙트와 함께 화면 중앙으로 플라이
2. **전투 이펙트**: 속성별 차별화된 파티클 (화염: 불꽃 파티클, 냉기: 얼음 결정, 번개: 전기 아크)
3. **시너지 발견**: 화면 중앙에 시너지 이름 + 속성 아이콘 합체 연출 (1.5초)
4. **데미지 숫자**: 팝업 + 바운스 + 페이드아웃 (크리티컬은 크기 2배 + 흔들림)
5. **보스 페이즈 전환**: 화면 암전 → 보스 변신 연출 → 새 BGM 시작
6. **카드 마스터리 진화**: 레벨업 시 카드 프레임이 변하는 연출 (은색→금색→애니메이션)
7. **이징**: 모든 UI 전환에 cubic-bezier(0.25, 0.1, 0.25, 1.0) — 부드럽고 자연스러운

### 엔진/프레임워크 추천

| 옵션 | 추천도 | 근거 |

| **Godot** | ★★★★☆ | 오픈소스, 경량, 2D 특화, 인디 개발에 적합. 다만 모바일 최적화 Unity 대비 약간 부족 |
| Cocos2d-x | ★★★☆☆ | 경량 2D 엔진이나 생태계 축소 중 |

**추천**: Unity 2D. 카드 드래그&드롭, 노드 맵, 전투 이펙트 등 복잡한 2D UI/인터랙션에 가장 적합.

### 핵심 기술 과제

1. **카드 시너지 엔진**: 200장+ 카드 × 시너지 조합의 실시간 검증/발동 시스템. 데이터 주도 설계 (JSON 테이블)
2. **프로시저럴 던전 생성**: 시드 기반 노드 맵 생성 + 규칙 검증 + 밸런싱 알고리즘
3. **적 AI 시스템**: 상태 기반 행동 패턴 테이블 + 보스 페이즈 시스템
4. **세이브/이어하기**: 턴 단위 자동 저장 + 앱 종료/전화 수신 등 모든 상황 대응
5. **카드 보상 밸런싱**: 현재 덱/시너지를 고려한 보상 풀 생성 알고리즘
6. **스트리머 모드 연동**: Twitch/YouTube API 연동, 실시간 투표 시스템
7. **애니메이션 성능**: 200장+ 카드의 개별 이펙트를 저사양 기기에서도 60fps 유지
