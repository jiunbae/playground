# Pocket Biome (포켓 바이옴) - 심화 기획문서

---

## 1. 게임 비전

### 한줄 소개 (엘리베이터 피치)
**"AI 생물들이 스스로 진화하는 주머니 속 생태계를 관찰하고, 이름 붙이고, 기록하는 관조형 시뮬레이션"**

### 게임의 핵심 감정
- **경이(Wonder)**: "이런 행동이 자연스럽게 발생하다니!" -- 이머전트 행동을 목격하는 순간의 놀라움
- **애착(Attachment)**: "봄이가 오늘 뭐 했을까?" -- 닉네임 붙인 개체에 대한 다마고치적 감정 연결
- **관조적 평온(Contemplative Peace)**: 자연 다큐멘터리를 보듯 생태계를 바라보는 명상적 고요함
- **발견의 기쁨(Joy of Discovery)**: 도감에 없던 희귀 행동을 목격하고 기록하는 탐험가적 성취감

### 디자인 철학
Pocket Biome은 "플레이어가 세상을 만드는 게임"이 아니라 **"세상이 스스로 살아가는 것을 바라보는 게임"**이다. 대부분의 시뮬레이션 게임이 플레이어의 조작과 최적화를 핵심으로 삼는 반면, 이 게임은 **관찰 그 자체가 플레이**이다. AI 에이전트의 단순한 규칙 조합이 예측 불가능한 드라마를 만들어내고, 플레이어는 그 드라마의 관객이자 기록자가 된다. 게임이 "해야 할 것"을 주는 대신, **"볼 것"과 "느낄 것"을 준다**.

닉네임/감정 아이콘 시스템으로 추상적 AI 에이전트를 "등장인물"로 전환하여, 관조적 깊이를 유지하면서도 캐주얼 유저까지 감정적으로 연결시키는 것이 R04의 핵심 설계 원칙이다.

### 타겟 유저 프로필

**1차 타겟**
- **한소율 유형 (30대 프리랜서/크리에이터)**: 인디 아트 게임 애호가. 관조적/실험적 경험을 사랑하며, 블로그/트위터에 관찰 일지를 연재물처럼 공유. 프리미엄 과금에 적극적. 예상 만족도: 47/50 (최고)
- **박준혁 유형 (30대 하드코어 게이머/개발자)**: AI 시스템을 분석하고 최적 초기 조건을 연구. 개입 영향 데이터 시각화를 A/B 테스트 프레임워크로 활용. 커뮤니티에 분석 공략 게시. 예상 만족도: 39/50

**2차 타겟**
- **이서연 유형 (20대 직장인)**: 출퇴근길 힐링 관찰, 인스타 스토리용 예쁜 스크린샷 공유. 닉네임 시스템으로 캐릭터에 정 들임. 예상 만족도: 42/50
- **정미경 유형 (40대 주부)**: 닉네임 붙인 생물을 아이와 함께 관찰. 카카오톡 카드로 관찰 일지 공유. 가이드 관찰 모드로 접근성 확보. 예상 만족도: 32/50
- **김하율 유형 (10대 학생)**: 대멸종/의외의 진화 등 극적 순간의 틱톡용 15초 타임랩스 공유. 예상 만족도: 32/50

**비타겟 (의도적 미타겟팅)**
- **최동현 유형 (20대 경쟁형 게이머)**: 경쟁/액션/리더보드 부재가 근본적 장벽. 주간 커뮤니티 이벤트로 최소한의 소셜 동기 부여하되, 이 유저를 위해 게임의 관조적 본질을 훼손하지 않음. 예상 만족도: 19/50

## 2. 코어 메커니즘 상세

### 핵심 동사 (플레이어가 하는 행동)
1. **관찰하다 (Observe)**: 생태계를 바라보고, 개체를 추적하고, 행동 패턴을 발견한다
2. **기록하다 (Record)**: 발견한 행동을 도감에 등록하고, 관찰 일지를 확인/편집한다
3. **이름 짓다 (Name)**: 개체에 닉네임을 붙여 감정적 연결을 형성한다
4. **개입하다 (Intervene)**: 관리자 모드에서 먹이/울타리/씨앗으로 생태계에 영향을 준다 (선택적)
5. **공유하다 (Share)**: 관찰 일지, 타임랩스, 캐릭터 카드를 SNS에 공유한다

### 조작 방식 (터치 제스처별 매핑)

| 제스처 | 기능 |
|--------|------|
| **탭 (개체)** | 개체 선택 → 닉네임/감정 아이콘/상태 표시, 추적 모드 진입 |
| **탭 (빈 공간)** | 선택 해제 / 관리자 모드에서 개입 지점 선택 |
| **핀치 줌** | 생태계 확대/축소 (미시/거시 관점 전환) |
| **드래그 (2손가락)** | 맵 패닝 (생태계 탐색) |
| **길게 누르기 (개체)** | 닉네임 변경 / 개체 전기(Biography) 열기 |
| **길게 누르기 (지형)** | 관리자 모드: 개입 메뉴 (먹이/울타리/씨앗/날씨) |
| **스와이프 위** | 관찰 일지 패널 열기 |
| **스와이프 아래** | 미니맵 / 도감 퀵 접근 |
| **더블 탭** | 해당 위치로 빠르게 줌인 |

### 난이도 곡선 설계

```
감정적 몰입도
^
|                                    ★ 세대 간 서사 형성
|                              ★ 대멸종/진화 목격
|                        ★ 관리자 모드 실험
|                  ★ 닉네임 캐릭터 애착 형성
|            ★ 첫 희귀 행동 발견
|      ★ 첫 관찰 일지 읽기
| ★ 가이드 관찰 모드 첫 실행
+──────────────────────────────────→ 시간
  Day 1   Day 3   Week 1  Week 2  Month 1  Month 3+
  (온보딩) (친숙해짐) (주기적) (관리자) (장기서사) (마스터리)
```

- **Day 1 (온보딩)**: 가이드 정령이 3분 내 핵심 안내. 지형 배치 → 생물 선택 → 첫 관찰. 가이드 관찰 모드 자동 활성화
- **Day 2~3**: 첫 관찰 일지 확인, 닉네임 붙이기, 푸시 알림 경험. "봄이가 뭐 했지?" 호기심 형성
- **Week 1**: 첫 희귀 행동 발견, 도감 수집 시작. 관찰자 모드와 가이드 모드 전환 익숙해짐
- **Week 2~3**: 관리자 모드 해금/체험. 개입의 영향 데이터 확인. 실험적 개입 시작
- **Month 1+**: 세대 유전/진화 트리 성장. 닉네임 캐릭터의 자손 추적. 관찰 일지 연재 콘텐츠화
- **Month 3+**: 주간 커뮤니티 이벤트 참여. 복수 바이옴 운영. 커뮤니티 도감 협력 완성

### 세션 설계

| 구분 | 시간 | 활동 |
|------|------|------|
| **최소 세션** | 2~3분 | 푸시 알림 → 리캡 확인 → 관찰 일지 읽기 → 닫기 |
| **최적 세션** | 8~15분 | 리캡 → 관찰 일지 → 관찰/탐험 → 도감 기록 → 공유 |
| **최대 세션** | 30~45분 | 관찰 + 관리자 모드 개입 + 데이터 분석 + 미니게임 + 주간 이벤트 |
| **딥 다이브 세션** | 1시간+ | 새 바이옴 생성, 진화 트리 분석, 커뮤니티 도감 기여, 블로그 콘텐츠 작성 |

## 3. 게임 모드별 상세

### 모드 1: 관찰자 모드 (기본/핵심)
- **목적**: AI 에이전트의 이머전트 행동을 순수하게 관찰하는 관조적 경험
- **규칙**: 환경 개입/미니게임 비활성화. 관찰, 추적, 도감 기록, 닉네임 편집만 가능
- **승리 조건**: 없음 (목표 기반이 아닌 경험 기반)
- **타겟 페르소나**: P6 한소율 (관조적 몰입), P2 이서연 (힐링 관찰)

### 모드 2: 가이드 관찰 모드 (관찰자 모드 파생)
- **목적**: 초보 유저에게 "지금 볼 만한 것"을 부드럽게 안내
- **규칙**: 관찰자 모드와 동일하나, 화면 하단에 가이드 정령의 소곤소곤 힌트 표시. "지금 동쪽 숲에서 흥미로운 일이 벌어지고 있어요" 등. 힌트는 무시 가능, 자동 소멸
- **승리 조건**: 없음
- **타겟 페르소나**: P4 정미경 (초보 진입), P1 김하율 (빠른 재미 발견)

### 모드 3: 관리자 모드 (서브)
- **목적**: 생태계에 능동적으로 개입하고 그 영향을 과학적으로 분석
- **규칙**: 일일 개입 5회 가능 (먹이 던지기/울타리 설치/씨앗 뿌리기/날씨 조절). 자연재해 시 미니게임 개입 가능 (30초~1분). 개입 전/후 행동 변화 데이터 시각화 제공
- **승리 조건**: 일일 미션 3개 완료 (관찰/개입 미션)
- **타겟 페르소나**: P3 박준혁 (실험/분석), P4 정미경 (먹이 주기 등 직관적 참여)

### 모드 4: 주간 커뮤니티 관찰 이벤트 (보너스)
- **목적**: 전체 유저가 동일 시드 바이옴을 관찰하고 발견을 공유하는 소셜 레이어
- **규칙**: 매주 동일 시드 배포. 관찰 모드 전용. 발견한 행동을 커뮤니티에 제출. 가장 많은 공감을 받은 발견이 "이번 주의 발견"으로 선정
- **승리 조건**: 없음 (경쟁이 아닌 "함께 발견하는 재미". 순위 대신 커뮤니티 도감 협력 완성)
- **타겟 페르소나**: P3 박준혁 (분석 능력 발휘), P6 한소율 (커뮤니티 참여), P1 김하율/P4 정미경 (우연한 발견의 가치)

### 모드 간 계층

```
[핵심] 관찰자 모드 ──파생──→ [핵심] 가이드 관찰 모드
          |
       전환 가능
          |
       [서브] 관리자 모드
          |
       [보너스] 주간 커뮤니티 이벤트 (관찰 모드 기반)
```

## 4. 콘텐츠 시스템

### 프로시저럴 생성 알고리즘 설계

**1. 이머전트 행동 시스템**
- 각 AI 에이전트: FSM(유한 상태 머신) + Behavior Tree(행동 트리) 구동
- 내부 상태 변수: 배고픔, 공포, 번식 욕구, 영역 본능, 사회성, 에너지
- 상태 변수 간 상호작용 규칙 → 예측 불가능한 집단 행동 발생
- 행동 감지 규칙: 포식/이동/번식/사망/무리형성/영역전쟁/공생 등 200+개 패턴 감지기

**2. 세대 유전 알고리즘**
- 유전 파라미터: 속도, 크기, 감각 범위, 색상, 공격성
- 번식 시: 양 부모의 파라미터를 교차(Crossover) + 소확률 돌연변이(Mutation)
- 세대 축적 → 환경 적응형 형질 자연 선택 → 진화 트리 자동 구축

**3. 환경 변화 알고리즘**
- 계절 순환: 날짜 기반 자동 전환 (봄-번식기, 여름-풍요기, 가을-이동기, 겨울-생존기)
- 자연재해: 확률적 발생 (가뭄 3%, 홍수 2%, 산불 1.5%, 전염병 1% / 일)
- 지형 변화: 침식, 식생 확장/축소, 수위 변화

**4. 오프라인 시뮬레이션**
- 시드 기반 결정론적 계산으로 오프라인 시간 경과 처리
- 7일 이상 미접속 시 주요 이벤트만 요약 처리 (성능 최적화)
- 리캡 타임랩스: 핵심 이벤트 30초 자동 편집

**5. 관찰 일지 자동 생성**
- 이벤트 감지 규칙 (포식/이동/번식/사망 등) → 우선순위 알고리즘 → 내러티브 템플릿 조합
- 닉네임 시스템 반영: "봄이가 새끼 3마리를 낳았다" 형태
- 이벤트 빈도 낮은 날: 개체 성장/환경 변화 등 소소한 기록으로 보완
- "오늘 아무 일도 없었다" 상황 최소화를 위한 이벤트 밀도 하한선 설정

**6. 닉네임 자동 생성**
- 형용사 풀(100+) + 동물명 풀(종별) + 이름 풀(200+)
- 조합 예: "용감한 토끼 봄이", "외로운 늑대 달이", "느긋한 사슴 하늘이"
- 개체 성격 파라미터(공격성, 사회성, 호기심)에 따라 형용사 매칭

### 콘텐츠 수량 계획

| 시점 | 바이옴 환경 | 생물 종 | 도감 행동 패턴 | 환경 테마 | 이벤트 |
|------|------------|---------|---------------|-----------|--------|

### 수집 요소

**1. 행동 도감**

**2. 캐릭터 앨범 & 가계도**
- 닉네임 붙인 개체의 일생 자동 기록 (탄생 → 주요 사건 → 사망)
- 세대별 가계도 트리 시각화
- "봄이의 손녀 꽃이가 알파가 됐다" 류의 세대 간 서사

**3. 생태계 아카이브**
- 각 바이옴의 역사 기록 (일별/주별/월별)
- "화산섬의 500일", "얼음 호수의 겨울" 등 각 바이옴이 하나의 이야기

**4. 관찰 일지 아카이브**
- 매일 자동 생성 일지 누적
- 주간/월간 "생태계 연대기" 자동 편집

**5. 커뮤니티 도감**
- 주간 이벤트에서 발견된 행동 누적
- 개인 도감과 별도, 전체 커뮤니티 협력 완성

**6. 개입 분석 리포트**
- 관리자 모드 개입 이력 + 결과 누적 분석
- "나의 개입 성향" (보호적/실험적/방관적)
- "가장 효과적이었던 개입", "의도치 않은 결과 Top 5"

## 10. 기술 요구사항

### 엔진/프레임워크 추천

| 영역 | 기술 | 선택 이유 |
|------|------|-----------|
| **게임 엔진** | Unity 2D (URP) | ECS/DOTS로 대량 에이전트 병렬 처리. 크로스 플랫폼 |
| **AI 시스템** | DOTS + Behavior Tree (NPBehave) | 수백 개 에이전트 효율적 병렬 처리 |
| **물리/이동** | Job System + Burst Compiler | 모바일에서 대규모 시뮬레이션 성능 |
| **오프라인 계산** | 시드 기반 결정론적 시뮬레이션 | 오프라인 시간 경과를 빠르게 재현 |
| **렌더링** | SpriteRenderer + GPU Instancing | 수백 개체 동시 렌더링 |
| **데이터 저장** | SQLite + JSON | 복잡한 생태계 상태 효율적 직렬화 |
| **타임랩스** | 프레임 버퍼 캡처 + AVFoundation/MediaCodec | 인앱 타임랩스 영상 생성 |
| **푸시 알림** | Firebase Cloud Messaging (FCM) | 오프라인 이벤트 기반 로컬 알림 |
| **분석** | Firebase Analytics | 유저 행동/밸런스 데이터 수집 |

### 핵심 기술 과제

**1. 대량 AI 에이전트 모바일 최적화**
- 문제: 200~500개 에이전트를 모바일에서 60fps로 구동
- 해결: DOTS(ECS) 기반 데이터 지향 설계. LOD 개념을 AI에 적용 (근거리: 풀 행동 트리, 원거리: 간략 상태 머신, 화면 밖: 간략화 로직). Job System + Burst Compiler로 병렬 처리

**2. 오프라인 시뮬레이션 정합성**
- 문제: 오프라인 기간의 시뮬레이션 결과가 온라인과 일관되어야 함
- 해결: 결정론적 시드 기반 계산. 고정 소수점 연산으로 플랫폼 간 동일 결과 보장. 7일 이상 미접속 시 주요 이벤트만 요약 처리

**3. 이머전트 행동 감지 시스템**
- 문제: AI 에이전트의 자발적 행동 중 "의미 있는" 이벤트를 자동 감지
- 해결: 행동 패턴 감지기(200+개 규칙). 이벤트 우선순위 알고리즘. 희귀 이벤트 트리거 확률 조정. 초기 "재미 보장 구간" 시드 튜닝

**4. 관찰 일지 내러티브 자동 생성**
- 문제: 이벤트 데이터를 자연스러운 한국어 내러티브로 변환
- 해결: 이벤트 감지 → 우선순위 정렬 → 내러티브 템플릿 조합 + 닉네임 삽입. 템플릿 풀 500+개로 반복감 최소화

**5. 주간 이벤트 시드 배포**
- 문제: 서버리스 원칙을 유지하면서 전체 유저 동일 시드 배포
- 해결: 주 1회 시드를 경량 CDN에서 배포 (월 $10 이내). 발견 공유는 Game Center/Google Play Games API 활용

### 오프라인/서버리스 아키텍처

```
[로컬 디바이스]
  ├── AI 시뮬레이션 엔진 (100% 로컬)
  ├── 오프라인 시뮬레이션 (시드 기반 결정론적)
  ├── 도감/관찰일지/캐릭터앨범 (SQLite)
  ├── 설정/프로필 (SharedPreferences)
  └── 타임랩스 생성 (로컬 렌더링)

[경량 서버 (최소)]
  ├── 주간 이벤트 시드 배포 (CDN, 주 1회)
  ├── 커뮤니티 발견 공유 (Game Center/Google Play Games)
  └── FCM 토큰 관리 (로컬 알림 스케줄링용은 서버 불필요)
```

### 데이터 저장 구조 (로컬)


### 대응 방안 우선순위

1. **(최우선)** 이머전트 행동 재미 보장 → 클로즈드 베타 집중 테스트
2. **(우선)** 모바일 성능 최적화 → DOTS/Burst 기반 아키텍처 조기 PoC
3. **(중요)** 과금 전환율 확보 → 소프트 론칭에서 A/B 테스트
4. **(관찰)** 니치 시장 한계 → 캐주얼 유저 KPI 모니터링, 필요 시 가이드 모드 강화
